# Project `.agent/skills` Policy

이 폴더는 이제 프로젝트 공통 설계 규칙의 source of truth가 아니다.

## 원칙

- 공통 아키텍처, 도메인, 워크플로 규칙은 `AGENTS.md`와 `agents/`에서 관리한다.
- `.agent/skills`는 에이전트 보조용 얇은 스킬만 남긴다.
- 프로젝트 규칙과 중복되는 내용은 이 폴더에 자세히 쓰지 않는다.

## 유지 방향

- 유지: 에이전트 소통 태도, 터미널 보조, 서버 관리 보조 같은 툴 보조 스킬
- 축소: 구조, API, 게임 설계, OSM 레이어, 코딩 스타일처럼 프로젝트 규칙과 겹치는 스킬

## 읽는 순서

1. `AGENTS.md`
2. 관련 `agents/` 문서
3. 필요할 때만 `.agent/skills`의 보조 스킬
