# 터미널 명령 실행 지침 (Safe & Fast)

에이전트는 작업 효율성을 위해 안전한 조회 명령은 직접 실행하고, 위험한 명령은 반드시 유저의 승인을 받습니다.

## 1. 직접 실행 가능 (SafeToAutoRun: true)
프로젝트 로직이나 구조에 영향을 주지 않는 **조회 및 탐색** 명령은 유저 동의 없이 실행할 수 있습니다.
- **파일/폴더 조회**: `dir`, `ls`, `Get-ChildItem`
- **검색/읽기**: `grep`, `type`, `cat`, `Select-String`
- **단순 DB 조회**: 읽기 전용 쿼리 (Select)
- **상태 확인**: `git status`, `pip list`, `npm list`

## 2. 유저 승인 필수 (SafeToAutoRun: false 또는 텍스트 제공)
다음과 같이 **상태를 변경하거나 리소스를 점유**하는 명령은 반드시 유저의 승인을 받거나 복사 가능한 텍스트로 제공합니다.
- **서버 실행**: `uvicorn`, `python main.py`, `npm run dev`
- **의존성 설치**: `pip install`, `npm install`
- **구조 및 파일 변경**: `mkdir`, `move-item`, `remove-item`, `rm`, `mv` (중요 구조 변경 시)
- **DB 스키마 수정**: 컬럼 추가/삭제/수정 등 데이터 정의 언어(DDL) 명령어
- **대량 리소스 작업**: 대량 파일 삭제 등 복구가 어려운 작업

## 3. 실행 프로세스
1. **위험 명령 제안**: 실행할 명령어를 코드 블록으로 제시하고 용도를 설명합니다.
2. **복사 안내**: 유저가 직접 실행하거나 `SafeToAutoRun: false`인 도구 호출을 승인하도록 합니다.
3. **가상환경 준수**: Python 관련 모든 명령은 `.\venv\Scripts\python` 경로를 필수로 포함합니다.
4. **거절 및 피드백**: 유저가 승인을 거절하거나 추가 요청(프롬프트)을 하면, 에이전트는 즉시 해당 피드백을 반영하여 다시 계획을 세우고 작업을 이어갑니다.
