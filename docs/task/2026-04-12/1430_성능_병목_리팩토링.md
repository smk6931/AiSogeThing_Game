# 성능 병목 리팩토링

## 작업 배경

파티션/몬스터 시스템 전반에서 플레이어 정체 시에도 계속 계산이 발생하는 패턴을 제거.
렉/병목의 주요 원인 9개 수정.

## 수정 파일 목록

### 프론트엔드

1. **`front/src/hooks/useCurrentRegionInfo.js`**
   - 25m 미만 이동 시 GPS state 갱신도 생략 (렌더 트리거 제거)

2. **`front/src/entity/world/RpgWorld.jsx`**
   - 1000ms 인터벌에 5m 이동 감지 추가 → 정체 중 polygon 연산 스킵
   - 인터벌 주기 1000ms → 1500ms
   - `visibleMonsterElements` useMemo에서 `debugConfig.playerScale` 의존성 제거
     (monsterScaleRef로 분리 — 디버그 설정 변경이 전체 몬스터 재계산 유발 방지)
   - 서버가 반경 내 몬스터만 전송하므로 클라이언트 100m 컬링 제거

3. **`front/src/entity/world/PartitionBoundaryOverlay.jsx`**
   - useEffect deps를 `[currentDong?.id]`만으로 단순화
   - 디버그 옵션(visibleMicro, visibleGroup, highlightCurrentGroup) 토글 시 API 재요청 방지

4. **`front/src/entity/world/GroupColorOverlay.jsx`**
   - 동일: useEffect deps를 `[currentDong?.id]`만으로 단순화
   - `anyVisible` 토글 시 재요청 방지

5. **`front/src/hooks/useAutoFarm.js`**
   - `currentTargetRef` 추가: 현재 타겟이 살아있고 범위 내에 있으면 전체 몬스터 순회 스킵
   - 자동사냥 ON 상태에서 1.5초마다 O(n) 재탐색 → 타겟 유효 시 O(1)

### 백엔드

6. **`back/monster/managers/MonsterManager.py`**
   - `_normal_count` 캐시 추가: 스폰/삭제 시점에만 갱신, 게임루프 `sum()` 제거
   - `math.sqrt()` 제거: distSq 비교로 교체 (`d_sq <= DETECT_RANGE_SQ`)
   - `nearest_dist > 3` → `nearest_dist_sq > 9` (일관성 유지)

7. **`back/world/services/partition_service.py`**
   - `get_current_region_info`: 기존 DB 이중 조회 제거 (partition_rows 한 번만 로드)
   - `_shape_cache` 딕셔너리 추가: partition id 기준으로 Shapely shape 객체 캐싱
     (반복 JSON 파싱 + `shape()` 생성 비용 제거)

8. **`back/player/routers/router.py`**
   - 몬스터 sync 이동 임계값 15m → 25m
   - 시간 임계값 2.0s → 3.0s

## 검증 방법

- 브라우저 Network 탭: `/region_info` 호출 빈도 확인 (이전보다 감소해야 함)
- 플레이어 5분 정체 시 CPU 사용량 이전 대비 확인
- 자동사냥 ON: 첫 타겟 후 콘솔에 불필요한 retarget 로그 없어야 함

## 남은 리스크

- `PartitionBoundaryOverlay`/`GroupColorOverlay`가 같은 dong의 데이터를 여전히 각자 fetch
  → 다음 단계: 상위 컴포넌트에서 단일 fetch 후 prop 전달
- `_shape_cache`는 서버 재시작 전까지 메모리에 유지 (동 수가 많아지면 관리 필요)
- `math` import는 MonsterManager에서 이동 AI에서 여전히 사용 중 (제거 불가)
