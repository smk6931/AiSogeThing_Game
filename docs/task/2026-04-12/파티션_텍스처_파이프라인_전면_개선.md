# 파티션 텍스처 파이프라인 전면 개선

## 작업 배경

- 파티션 텍스처 2차 결과물에서 두 가지 문제 확인
  1. RESIDENTIAL 등 그룹 프롬프트가 건물 내부/지붕 장면을 생성
  2. polygon 외부가 검정으로 남는 문제
- VAEEncodeForInpaint 방식의 ring border artifact 문제
- 모델 퀄리티가 참조 이미지(`ds_lava_canyon` 등) 대비 낮음

## 주요 변경사항

### 1. DB 컬럼 정리 (`world_partition`, `world_partition_group`)

삭제 컬럼:
- `world_partition`: `persona_tag`, `source_feature`, `gameplay_meta`, `image_prompt_append`, `resolved_prompt`, `resolved_prompt_negative`, `prompt_resolved_at`
- `world_partition_group`: `persona_tag`, `image_prompt_negative`, `prompt_inherit_mode`

의존 뷰 재생성: `world_level`, `world_level_partition`, `world_partition_detail` (CASCADE DROP → 재생성)

### 2. 워크플로우 전환 (핵심)

**이전**: `VAEEncodeForInpaint + polygon mask → 단일 패스 1024px`
- ring border artifact (mask 경계에 AI가 석재 테두리 생성)
- 낮은 퀄리티

**이후**: `EmptyLatentImage → hi-res fix(1.5x) → 4xUltrasharp → PIL polygon clip`
- Pass 1: STEPS=8, CFG=2.0, dpmpp_sde, denoise=1.0
- Pass 2: 1.5x 업스케일 → re-encode → denoise=0.5 (hi-res fix)
- Pass 3: 4xUltrasharp 업스케일 → 2× 최종 출력
- PIL post-process: polygon 외부 검정 마스킹 (feather=6)
- ring border 완전 제거, 퀄리티 대폭 향상

원인 파악: 동일 모델(`dreamshaperXL_lightningDPMSDE.safetensors`)로 참조 이미지 생성이 가능했고 차이는 4xUltrasharp 업스케일러 사용 여부였음

### 3. 프롬프트 시스템 단순화

- Negative 프롬프트 재활성화: `cartoon, anime, flat colors, isometric, building interior, rooftop view...`
- 그룹 `image_prompt_base` 만 사용 (partition별 persona 제거)
- `--override-prompt` CLI 옵션 추가 (테스트용)
- `STYLE_PREFIX` + `STYLE_NEGATIVE` 상수로 관리

### 4. SQL 쿼리 정리

`run_partition_mode()` 쿼리에서 삭제된 컬럼 참조 제거:
- `p.persona_tag`, `p.image_prompt_append`, `p.image_prompt_negative`
- `g.image_prompt_negative`, `a.image_prompt_base`, `a.image_prompt_negative`
