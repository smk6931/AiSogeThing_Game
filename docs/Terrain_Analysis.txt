1. 현재 레이어 구조 (Architecture)
현재 시스템은 6개 이상의 독립적인 레이어를 Y축(고도) 순서대로 쌓아 올리는 계단식 적층 구조입니다.

- Layer 0 (MapTiles): 기본 2D 지도 타일
- Layer 1 (DongGroundMesh): 동(Dong) 경계 기반의 기본 바닥 (Green Flat Mesh)
- Layer 2 (SeoulTerrain): 숲, 공원, 주요 도로 등 고정된 자연/도로 레이어
- Layer 3 (CityBlockOverlay): 도로 중심의 블록 텍스처
- Layer 4 (ZoneOverlay): 실시간 OSM 기반 용도구역 (주거, 상업 등)
- Layer 5 (SeoulHeightMap): 등고선 기반 3D 지형

2. 분석 결과 (Technical Findings)

--- 장점 (Pros) ---
- 모듈성: 각 레이어가 독립적으로 로드 및 표시되므로 유지보수가 용이함.
- 실제 데이터 기반: OSM 데이터를 직접 사용하여 서울의 실제 구성을 정확히 반영함.
- LOD 스트리밍: 동(Dong) 단위로 데이터를 쪼개어 플레이어 주변만 정밀하게 로드함.

--- 단점 및 한계 (Cons & Gaps) ---
- 시각적 빈 공간 (Coverage Gaps): 
  - OSM 데이터에서 특정 카테고리가 정의되지 않은 구역은 레이어가 생성되지 않음.
  - 이 경우 하위 레이어의 단순한 단색 바닥이 드러나며 RPG 데이터가 빈 느낌을 줌.
- 중첩 및 성능 부하 (Overlap & Overdraw):
  - 수많은 레이어가 겹쳐 있어 렌더링 부하(Draw Calls)가 높음.
  - 레이어 간 고도 차이가 미세하여 Z-Fighting 발생 위험이 있음.
- 자연스러운 블렌딩 부재:
  - 숲과 도로 경계가 칼같이 잘려 RPG 대지의 느낌이 부족함.

3. RPG 바닥 텍스처를 위한 개선 제안

1. 도로망 기반 부지 자동 분할 (Sector Plots) 활성화
- 개별 동(Dong) 경계를 도로망으로 한 번 더 쪼갠 '섹터' 레이어를 전면에 배치.
- 데이터가 누락된 구역도 도로가 있다면 자동으로 '블록'이 생성되도록 하여 100% 면적 커버 달성.
- CityBlockOverlay의 비활성화된 섹터 렌더링 기능을 개선하여 개별 텍스처링 적용.

2. 통합 셰이더 기반 Splat Map 도입
- 독립된 메시 레이어를 줄이고 하나의 베이스 메시 위에서 셰이더가 여러 텍스처를 합성하도록 변경.

2. Splat Map 생성
OSM 데이터를 읽어 속성(풀, 흙, 아스팔트)을 R, G, B 채널에 기록한 가상의 맵 사용.

3. Stochastic Blending
셰이더에서 노이즈를 섞어 텍스처 간 경계를 부드럽게 뭉개어 자연스러운 지형 표현.

4. Fallback Detail
데이터가 없는 지역도 "기본 흙+잔디" 텍스처가 깔리도록 하여 빈 공간 느낌 제거.

4. 실행 계획 (Implementation Summary)
1. 백엔드: 동 단위 데이터 응답 시 Splat 인덱스 미리 계산 (선택 사항).
2. 프론트엔드: DongGroundMesh에 멀티 텍스처 셰이더 적용.
3. 프론트엔드: 레이어 중복 데이터 정리 (Terrain vs Zone vs Block 통합).
