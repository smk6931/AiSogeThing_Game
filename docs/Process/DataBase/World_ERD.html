Current Structure

world_admin_area는 행정 계층, world_level_partition는 실제 플레이 셀을 담당합니다. 현재 구조에서는 파티션 연결 테이블 없이 자유 이동과 좌표 판정 중심으로 운영합니다.

WORLD_ADMIN_AREA
서울 / 구 / 동 계층

  • osm_id
  • area_level
  • area_code
  • name
  • name_en
  • parent_id
  • center_lat
  • center_lng
  • boundary_geojson
  • area_meta
WORLD_LEVEL_PARTITION
가장 작은 플레이 파티션
  • id
  • partition_key
  • admin_area_id
  • partition_stage

  • partition_type
  • map_name
  • display_name

  • group_key
  • group_display_name
  • group_theme_code
  • landuse_code

왜 2개로 두는가 현재는 오픈월드 자유 이동 구조라서, 위치 판정과 상위 행정 계층만 먼저 유지합니다. 연결 그래프는 나중에 런타임 길찾기로 대체할 수 있습니다.
group_*의 현재 의미 별도 테이블은 아니고, 여러 micro partition을 하나의 플레이 지역처럼 보이게 묶는 메타데이터입니다.

Recommended Structure For Codex

도감, 퀘스트, 몬스터 서식지, 채집 구역 설명까지 가려면 world_region_group 같은 플레이 지역 테이블을 추가하는 편이 더 깔끔합니다.

도감 기준 엔티티 도감은 partition보다 region group 단위를 보여주는 편이 자연스럽습니다. 파티션은 너무 잘게 쪼개져 있어 설명 단위로 부적합합니다.
실무 분리 현재 위치 판정은 partition, 상위 지역 설명은 region group, 주소 계층은 admin area가 맡는 구조가 가장 안정적입니다.
