# 3D 지형 고도 설계 (통합 문서)

> 현재 방향: 파티션 단위 계단식 고도 (Option B)
> 2.5D 카메라/이동 시스템과 완전 호환 — 공간감 강화 목적
> 참고 레퍼런스: hordes.io (하이트맵 + Fragment Shader 등고선)
> 채택 결정일: 2026-04-11

---

## 1. 배경 및 문제

현재 게임 월드는 완전 평면(`y=0.55 고정`) — 공간감과 지형 느낌 없음.

```
CityBlockOverlay.jsx
  buildTerrainBlock() → positions[vi++] = 0.55  // 완전 고정 평면
```

파티션 시스템이 OSM 경계 기반으로 이미 잘 쪼개져 있어 고도 도입의 기반이 됨.

---

## 2. 선택지 비교

### Option A — hordes.io 방식 전면 3D 지형

```
장점: 비주얼이 hordes.io급, smooth terrain
단점:
  - 플레이어 이동/충돌 전면 재설계
  - 몬스터 스폰 로직 변경
  - 카메라 terrain-follow 추가
  - 국토지리정보원 DEM 데이터 파이프라인 신규 구축
  - 파티션 polygon → 3D 지형 위 draping 새로 설계
  - 최소 2~3주 작업, 기존 코드 상당 부분 리셋
  - 2.5D 방향과 충돌
```

### Option B — 파티션 단위 계단식 고도 (채택)

```
장점:
  - CityBlockOverlay.jsx 최소 수정 (y-offset만 추가)
  - 플레이어/몬스터/카메라 코드 변경 없음
  - 2.5D 방향 완전 호환, 공간감 강화
  - 실제 서울 DEM 평균값 사용 가능
  - v2 smooth terrain 전환 시 elevation 데이터 그대로 활용
단점:
  - Smooth terrain 아님 (계단식)
  - 등고선 표현 제한적
  - 파티션 경계에서 단차 발생 → Phase 3에서 옹벽/경사면으로 완화
```

**결정: Option B 채택.**

---

## 3. 목표 구조

```
파티션별 elevation_m (DB)
  → CityBlockOverlay에서 y = 0.55 + elevation_m * ELEV_SCALE
  → 서울 지형의 대략적 기복이 월드에 반영됨
```

서울 고도 범위:

| 지역 | 실제 고도 |
|---|---|
| 한강 수면 | ~0m |
| 강남/여의도 평지 | 10~25m |
| 강북 구릉지 | 50~150m |
| 북악산, 인왕산 | 200~340m |
| 관악산 | 629m |
| 북한산 | 836m |

게임 내 스케일 — 실제 고도 압축:

```
ELEV_SCALE = 0.003   // 100m 실제 = 0.3 world unit
→ 북한산(836m) = 2.5 unit 단차
→ 강남 평지(20m) = 0.06 unit 단차
```

## 4. DEM 데이터 파이프라인

### 데이터 소스

**1순위 — 국토지리정보원 (NGII) 오픈마켓**
- URL: https://map.ngii.go.kr/ms/pblictn/rsrchDatList.do
- 상품: 수치표고모델(DEM) 5m 해상도
- 포맷: GeoTIFF (EPSG:5186 또는 EPSG:4326)
- 서울 범위: 약 16장 타일 (1:5000 도엽)
- 라이선스: 공공데이터 개방 (비상업 무료, 상업 출처표기)

**2순위 — OpenTopoData SRTM (빠른 테스트용)**
- endpoint: `https://api.opentopodata.org/v1/srtm30m?locations=lat,lng`
- 배치 최대 100점 → 파티션 centroid 일괄 쿼리
- 무료 API 초당 1요청 제한 → 100점씩 묶어서 호출
- 30m 해상도 (정식 서비스 전 검증용)

### DB 마이그레이션

```sql
-- Alembic revision: add_elevation_m_to_partition
ALTER TABLE world_partition ADD COLUMN elevation_m FLOAT DEFAULT 0.0;
```

### 처리 스크립트 (`back/scripts/compute_partition_elevation.py`)

```python
import rasterio
import numpy as np
from shapely.geometry import shape
from core.database import fetch_all, execute

# 1. DEM GeoTIFF 로드
dem = rasterio.open('data/seoul_dem_5m.tif')

# 2. 파티션 boundary_geojson 로드
partitions = await fetch_all("SELECT partition_key, boundary_geojson FROM world_partition")

# 3. 파티션별 DEM 평균 고도 계산
for p in partitions:
    geom = shape(p['boundary_geojson'])
    out_image, _ = rasterio.mask.mask(dem, [geom], crop=True)
    valid = out_image[out_image != dem.nodata]
    elevation = float(np.mean(valid)) if len(valid) > 0 else 0.0

    await execute(
        "UPDATE world_partition SET elevation_m = :elev WHERE partition_key = :key",
        {"elev": elevation, "key": p["partition_key"]}
    )
```

## 5. 렌더링 적용

### CityBlockOverlay.jsx 수정

```jsx
// 현재
positions[vi++] = 0.55;

// 변경
const elev = partition.elevation_m ?? 0;
const gameY = 0.55 + elev * ELEV_SCALE;
positions[vi++] = gameY;
```

API 응답 `getDongPartitions`에 `elevation_m` 포함 필요.

### 경계 단차 완화

인접 파티션 간 고도 차가 크면 경계가 절벽처럼 보임.

**방법 A — 고도 캡 (간단, 우선 적용)**
```js
const MAX_ELEV_DIFF = 5; // m 단위
const cappedElev = Math.min(partition.elevation_m, neighborAvg + MAX_ELEV_DIFF);
```

**방법 B — 경사면 mesh (고퀄, Phase 3)**
- 파티션 경계 edge마다 인접 파티션 고도 차만큼 수직 quad 추가
- 각 edge 선분마다 `[x, yLow, z] ~ [x, yHigh, z]` 두 삼각형
- 텍스처: concrete / stone wall 재질

## 6. 구현 체크리스트

### Phase 1 — 데이터 준비
- [ ] `world_partition` 테이블에 `elevation_m` 컬럼 추가 (Alembic 마이그레이션)

  - `terrain_service._fetch_area_elevations()` 이미 구조 있음 — 활성화만 하면 됨
- [ ] 파티션별 고도값 DB 저장 (배치 스크립트)
- [ ] (장기) 국토지리정보원 DEM 5m로 교체 — 정밀도 필요 시

### Phase 2 — 렌더링 기본
- [ ] `CityBlockOverlay.jsx`: 파티션별 `elevation_m` 기반 y-offset 적용
- [ ] API 응답 `getDongPartitions`에 `elevation_m` 포함
- [ ] 인접 파티션 간 단차 캡 적용 (방법 A)

### Phase 3 — 파티션 경계 옹벽 처리
- [ ] 인접 파티션 쌍 목록 사전 계산 (bbox 겹침 또는 Shapely intersection)
  - 주의: adjacency 테이블이 drop된 상태 → bbox 겹침으로 재계산 필요
- [ ] 고도 차 > 임계값인 경계 edge 추출 (공유 경계선)
- [ ] 경계 edge를 따라 수직 quad(옹벽) mesh 생성
- [ ] 도로가 경계를 건너는 구간 → 옹벽 뚫기 (도로 폭만큼 gap)

### Phase 4 — 도로 경사 ramp
- [ ] OSM 도로(`highway`)가 파티션 경계를 건너는 선분 탐지
- [ ] 해당 도로 segment에 yLow → yHigh 점진 보간 geometry 적용
- [ ] 옹벽에서 도로 통과 부분은 gap으로 처리 (Phase 3 연동)

### Phase 5 — 플레이어 이동 전환
- [ ] 플레이어가 파티션 경계 넘을 때 현재 파티션 `elevation_m` 쿼리
- [ ] y 좌표를 `lerp(yPrev, yNext, t)`로 부드럽게 전환
- [ ] 몬스터/NPC도 동일 고도 기준 적용

### Phase 6 — 시각 개선 (선택)
- [ ] Fragment Shader 등고선 — `fract()` 기반 라인
- [ ] 고도별 텍스처 재질 가중치 변화 (산지=rock, 저지대=grass)

## 7. 주의사항

- 현재 `positions[vi++] = 0.55` (CityBlockOverlay) 이 값을 `0.55 + elevation` 으로 교체
- 고도 스케일 변환 필수: `ELEV_SCALE = 0.003` (100m → 0.3 world unit)
- 서울 고도 범위: 0m(한강) ~ 836m(북한산) — 게임에서는 압축 적용
- 파티션이 없는 구역은 기본 y=0 유지
- OpenTopoData 무료 API는 초당 1요청 제한 → 배치 스크립트에서 100점 묶어서 호출
- adjacency 테이블 drop 상태 → Phase 3 시 bbox 겹침으로 재계산 필요
- 옹벽 mesh는 렌더 시 동적 생성보다 빌드 타임 사전 계산이 성능상 유리

## 8. 미래 v2 — hordes.io 방식 전환 참고

현재 Option B에서 장기적으로 smooth terrain으로 전환할 때 쓸 기법.

### 하이트맵 Vertex Shader

```glsl
uniform sampler2D uHeightmap;
uniform float uMaxHeight;

void main() {
  vec2 uv = (position.xz / terrainSize) + 0.5;
  float h = texture2D(uHeightmap, uv).r * uMaxHeight;
  vec3 displaced = vec3(position.x, h, position.z);
  gl_Position = projectionMatrix * modelViewMatrix * vec4(displaced, 1.0);
  vHeight = h;
}
```

### 등고선 Fragment Shader

```glsl
varying float vHeight;
uniform float uContourInterval;

void main() {
  vec3 baseColor = texture2D(uTerrain, vUv).rgb;
  float c = fract(vHeight / uContourInterval);
  float lw = fwidth(vHeight / uContourInterval) * 1.5;
  float line = smoothstep(0.0, lw, c) * smoothstep(0.0, lw, 1.0 - c);
  vec3 contourColor = vec3(0.25, 0.18, 0.08);
  gl_FragColor = vec4(mix(contourColor, baseColor, line), 1.0);
}
```

### 텍스처 스플래팅 (고도·경사별)

```glsl
float grass = clamp(1.0 - abs(vHeight - 30.0) / 50.0, 0.0, 1.0);
float rock  = clamp((vSlope - 0.4) * 3.0, 0.0, 1.0);
float snow  = clamp((vHeight - 400.0) / 100.0, 0.0, 1.0);
vec3 color = grassTex * grass + rockTex * rock + snowTex * snow;
```

### LOD (Level of Detail)

```
카메라 위치 기준 동심원:
  0~200m: 256×256 그리드 (고해상도)
  200~500m: 64×64 그리드
  500m~: 16×16 그리드 (저해상도)
```
