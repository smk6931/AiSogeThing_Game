# [예정] 특수 지형 파티션 고도화

> **전제 조건**: `partition_texture_생성가이드.md` 기준으로 동작구 전체 동의 기본 파티션 생성이 완료되어 있어야 한다.
> 이 문서는 그 이후 추가 처리(수계 / 철도 / 녹지)를 다룬다.

---

## 1. 왜 추가 처리가 필요한가

기본 파티션(`build_partitions_from_osm.py`)은 **도로로 둘러싸인 블록** 단위로 쪼갬.
도로망이 명확한 주거/상업지에서는 잘 동작하지만, 아래 지형에선 부자연스러움.

| 지형 | 현재 결과 | 문제 |
|------|-----------|------|
| 한강 / 안양천 | 도로 삼각형 조각들 | 강이 이상한 다각형으로 분할됨 |
| 경부선 철도 | 인접 블록에 흡수 | 철도 자체가 파티션으로 존재하지 않음 |
| 공원 / 산림 | 도로 격자로 분할 | 공원이 4~6개 네모 조각으로 나뉨 |

---

## 2. 처리 순서

기본 파티션 생성 완료 후, 해당 지역의 기존 파티션을 **교체(DELETE → INSERT)** 하는 방식.

```
[완료된 기본 파티션]
    ↓
Step A: 수계 처리  (한강, 안양천, 도림천)
Step B: 철도 회랑 처리  (경부선/1호선 구간)
Step C: 대형 녹지 처리  (공원, 산림)
    ↓
regroup_partition_members.py 재실행
```

각 Step은 독립적으로 실행 가능. 한 번에 하나씩 적용.

---

## 3. Step A — 수계 (강 / 하천)

### 기준

```python
# OSM waterway 폴리곤을 bbox 방향으로 세그먼트 분할
WATERWAY_SEGMENT_M = 300   # 약 300m 단위로 분할
theme_code = 'ancient_waterway'
dominant_landuse = 'water'
is_road = False
```

### 분할 방식

강의 유속 방향(장축)에 **수직**으로 300m마다 자름:
- `longest_axis_split`을 반대 축으로 적용 (장축 방향으로 자르는 게 아닌 단축 방향)
- 또는 교량(OSM `bridge=yes`) 위치를 기준점으로 삼아 자연스러운 구간 결정

```
한강 노량진 구간 예시:
  ├─ hangang_s01: 노량진철교 서쪽 ~ 노량진철교
  ├─ hangang_s02: 노량진철교 ~ 한강철교
  └─ hangang_s03: 한강철교 ~ 동쪽 경계
```

### 적용 대상 (동작구)

| 수계 | 구간 | 예상 파티션 수 |
|------|------|---------------|
| 한강 | 노량진1동~노량진2동 북단 | 3~5개 |
| 안양천 | 노량진2동 서쪽 합류부 | 2~3개 |
| 도림천 | 노량진2동 관통 | 2~4개 |

### 스크립트 (예정)

```bash
python back/scripts/build_waterway_partitions.py --dong-osm-id 3879474
python back/scripts/build_waterway_partitions.py --dong-osm-id 3879477
```

## 4. Step B — 철도 회랑

### 기준

```python
# OSM railway=rail 라인 → buffer → 폴리곤
RAIL_BUFFER_M = 15   # 레일 양쪽 15m (= 선로 폭 약 30m)
theme_code = 'iron_rail_corridor'
dominant_landuse = 'industrial'
is_road = False
```

### 분할 방식

역(station) 위치를 기준으로 구간 분할:
```
노량진역 구내 (station polygon) → 별도 파티션 1개
역 사이 선로 구간 → 직선 buffer 폴리곤 1개/구간
```

### 적용 대상 (동작구)

| 노선 | 해당 동 | 예상 파티션 수 |
|------|---------|---------------|
| 경부선/1호선 | 노량진1동 북단 | 3~4개 |
| 지하철 9호선 | 노량진1동 | 지하 → 미적용 (지하선로는 지상 영향 없음) |

### 스크립트 (예정)

```bash
python back/scripts/build_railway_partitions.py --dong-osm-id 3879474
```

## 5. Step C — 대형 녹지 (공원 / 산림)

### 기준

```python
# OSM park/forest polygon 직접 사용
# 10,000 m² 초과 시에만 추가 분할
PARK_MAX_M2 = 10_000
theme_code = 'sanctuary_green'
dominant_landuse = 'park' | 'forest'
is_road = False
```

### 분할 방식

우선순위:
1. **내부 보행로(footway/path)가 있으면** → footway로 polygonize해서 자연스러운 구역 분할
2. **없으면** → `longest_axis_split` 기존 방식 유지

```
용봉정근린공원 예시:
  OSM polygon 추출
    ├─ 내부 footway 라인 추출
    ├─ footway로 polygonize
    └─ 구역마다 별도 파티션
       (산책로 따라 자연스럽게 나눈 느낌)
```

### 적용 대상 (동작구)

| 공원 | 동 | 면적 | 예상 파티션 수 |
|------|----|----|---------------|
| 용봉정근린공원 | 노량진1동 | ~55,000m² | 4~6개 |
| 노량진공원 | 노량진2동 | ~35,000m² | 3~5개 |
| 노량진수산시장 녹지 | 노량진2동 | - | 1~2개 |

### 스크립트 (예정)

```bash
python back/scripts/build_park_partitions.py --dong-osm-id 3879474
python back/scripts/build_park_partitions.py --dong-osm-id 3879477
```

## 6. 전체 적용 순서 (동작구 완성 기준)

```
Phase 1 — 기본 파티션 (현재 진행 중)
  ├─ 노량진1동 ✅
  ├─ 노량진2동 ✅
  ├─ 상도1~4동
  ├─ 사당1~4동
  ├─ 대방동
  └─ 신대방1~2동

Phase 2 — 수계/철도/녹지 추가 처리 [이 문서]
  ├─ 한강/안양천/도림천 수계
  ├─ 경부선 철도 회랑
  └─ 대형 공원/산림

Phase 3 — 고도(elevation) 적용 [3D지형_고도_설계.md 참조]
  └─ DEM → 파티션 elevation_m → CityBlockOverlay y-offset
```

---

## 7. 주의사항

- 추가 처리는 **기존 파티션을 교체**하므로 반드시 백업 먼저
- `regroup_partition_members.py`는 Step A/B/C 각각 완료 후 재실행
- 수계 파티션은 `is_road=false`이지만 `group_key`는 별도 수계 그룹으로 분리
- 철도 회랑은 `is_road=false`, 선로 자체는 보행 불가 → `gameplay_meta: {"walkable": false}`
