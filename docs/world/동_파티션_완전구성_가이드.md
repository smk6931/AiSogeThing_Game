# 동(洞) 파티션 완전 구성 가이드

> 새로운 동에 파티션을 처음부터 완성까지 설정하는 전체 파이프라인.  
> Phase 1 → Phase 2 → Phase 3 순서로 실행한다.  
> 이 파일 하나만 보면 된다. 각 Phase의 상세 설계 배경은 하단 "관련 문서" 참조.

## Phase 1 — 기본 파티션 생성 (도로 기반)

> 상세: [`partition_texture_생성가이드.md`](./partition_texture_생성가이드.md)

### 백업 (필수)

```bash
docker exec game-sogething-db pg_dump -U game_sogething -d game_sogething \
  -t world_partition -t world_partition_group -t world_partition_group_member \
  --data-only --inserts \
  > back/world/data/backups/partition_backup_{동이름}_$(date +%Y%m%d).sql
```

### 실행 순서

```bash
# 1. OSM feature 수집
python back/scripts/fetch_osm_features.py --dong-osm-id {OSM_ID}

# 2. 파티션 생성 dry-run (예상 파티션 수 / 면적 확인)
python back/scripts/build_partitions_from_osm.py --dong-osm-id {OSM_ID} --dry-run

# 3. 파티션 생성 (실행)
python back/scripts/build_partitions_from_osm.py --dong-osm-id {OSM_ID}

# 4. 그룹 재배정
python back/scripts/regroup_partition_members.py --dong-osm-id {OSM_ID}
```

```bash
# 5. gap 채우기 (동 경계와 파티션 union 사이 빈 영역 → 인접 파티션 흡수)
python back/scripts/fill_partition_gaps.py --dong-osm-id {OSM_ID} --dry-run
python back/scripts/fill_partition_gaps.py --dong-osm-id {OSM_ID}
```

### Phase 1 완료 기준

- [ ] `world_partition` 테이블에 파티션 삽입됨
- [ ] `world_partition_group_member`에 그룹 배정 완료
- [ ] `fill_partition_gaps.py` 실행 후 미처리 gap 없음 확인
- [ ] 인게임 GroupColorOverlay(그룹색 레이어)로 그룹 경계 확인

---

## Phase 2 — 특수 지형 처리 (수계 / 녹지 / 공원)

> 상세: [`[예정]_특수지형_파티션_고도화.md`](./%5B예정%5D_%특수지형_%파티션_%고도화.md)  
> Phase 1 완료 후 실행. 도로로 잘린 수계/공원/산림 파티션을 OSM 폴리곤 기준으로 교체.

### 실행

```bash
# 수계 + 공원 + 산림 특수 처리 (dry-run으로 먼저 확인)
python back/scripts/build_special_terrain_partitions.py --dong-osm-id {OSM_ID} --dry-run

# 실행
python back/scripts/build_special_terrain_partitions.py --dong-osm-id {OSM_ID}

# 특수 처리 후 그룹 재배정 재실행
python back/scripts/regroup_partition_members.py --dong-osm-id {OSM_ID}
```

### Phase 2 완료 기준

- [ ] 강/하천이 단일 폴리곤(또는 구간 분할)으로 바뀜
- [ ] 공원/산림이 도로 격자 조각이 아닌 하나의 덩어리로 바뀜
- [ ] 파티션 총 수 감소 확인 (ex: 285→230)
- [ ] 인게임 파티션 레이어에서 `dominant_landuse=water/park/forest` 확인

---

## Phase 3 — 그룹 boundary 업데이트 (바닥 렌더링용)

> 상세: [`docs/plan/2026-04-12/1600_바닥_텍스처_그룹_단위_렌더링.md`](../plan/2026-04-12/1600_바닥_텍스처_그룹_단위_렌더링.md)  
> Phase 2 완료 후 실행.  
> `world_partition_group.boundary_geojson`을 개별 파티션 모음 → Shapely unary_union 결과로 교체.

### 실행

```bash
# dry-run: 그룹별 결과 확인
python back/scripts/update_group_boundaries.py --dong-osm-id {OSM_ID} --dry-run

# 실행
python back/scripts/update_group_boundaries.py --dong-osm-id {OSM_ID}
```

### Phase 3 완료 기준

- [ ] `world_partition_group.boundary_geojson` 타입이 `Polygon` 또는 `MultiPolygon`으로 바뀜
- [ ] 인게임 바닥 텍스처가 도로 격자선 없이 그룹 단위로 하나의 면으로 렌더링됨
- [ ] 바닥 mesh 수 대폭 감소 (파티션 수 → 그룹 수 수준)
