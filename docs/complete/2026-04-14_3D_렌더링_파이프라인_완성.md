# 3D 렌더링 파이프라인 완성 (2026-04-14)

## 개요

Three.js 기반 서울 등고선 게임의 렌더링 구조를 2D 오버레이(painter's algorithm) 방식에서
진짜 3D 씬(GPU depth sorting) 방식으로 전환 완료.
약 3일간의 작업으로 카메라 360도 회전 시 절벽 깜박임, 구멍, Z-fighting 전부 해결.

---

## 문제 → 해결 흐름

### 1. 파티션 사이 절벽 빈 공간 (Sparse Cliff Gaps)

**문제**: 인접 파티션 경계 엣지 좌표가 GeoJSON 정밀도 문제로 미세하게 달라서
파티션 단위로 절벽을 생성하면 파티션 사이에 틈이 생겼다.

**해결**: `buildGroupBoundaryCliffs` → 그룹 union polygon 외곽 링 기반 압출.
파티션 단위 엣지 매칭 불필요. union polygon이 내부 갭 없는 깨끗한 경계 제공.

---

### 2. 카메라 대각선 시야 검정 void (Camera Angle Void)

**문제**: 카메라를 낮게 내려다보면 파티션 외곽 너머로 검정 void가 보였다.

**해결**: RpgWorld에 `showElevation ON` 시 Y=-1.8에 30km 광역 바닥 평면 추가.
```jsx
{showElevation && (
  <mesh rotation={[-Math.PI / 2, 0, 0]} position={[0, -1.8, 0]} renderOrder={0}>
    <planeGeometry args={[30000, 30000]} />
    <meshBasicMaterial color="#3a3020" depthWrite={true} />
  </mesh>
)}
```

---

### 3. 카메라 회전 시 흰색/베이지 영역 출현

**문제 1**: `EqualStencilFunc`이 고도가 올라간 바닥면(Y>0)에서 실패 → DongMask(Y=0)와 불일치.

**해결**: `showElevation ON` 시 `AlwaysStencilFunc`으로 전환.

**문제 2**: SeoulTerrain 도로 renderOrder=104~112가 elevated 파티션 위에 그려졌다.

**해결**: `showElevation ON` 시 도로 `renderOrder=2`로 내림.

---

### 4. SketchUp push/pull 방식 3D 압출 (buildExtrudedPolygon)

**문제**: 기존 방식은 파티션 높이만큼 타일을 쌓는 방식 → 옆에서 보면 구조 노출, 자연스럽지 않음.

**해결**: `buildExtrudedPolygon` 구현.
- `geometry.groups[0]` = 상단면 (파티션/테마 텍스처)
- `geometry.groups[1]` = 측면벽 (절벽 텍스처)
- 상단면 `side={THREE.DoubleSide}`: 카메라가 박스 내부를 향해도 top face 렌더 → black hole 방지
- `depthWrite={true}`: GPU depth sorting → 카메라 회전 시 절벽 깜박임 없음

```jsx
<mesh geometry={block.geo} renderOrder={block.order}>
  <meshBasicMaterial attach="material-0" map={topTex} side={THREE.DoubleSide} depthWrite={true} />
  <meshBasicMaterial attach="material-1" map={wallTex} color={wallColor} side={THREE.DoubleSide} depthWrite={true} />
</mesh>
```

### 5. 인접 그룹 공유 경계 Z-fighting (핵심 버그)

**문제**: 그룹 A와 그룹 B가 맞닿는 경계선에 각 그룹이 벽을 독립적으로 생성.
같은 위치에 벽 2개 → 카메라 각도에 따라 어떤 면이 앞인지 GPU 판단이 매 프레임 달라짐 → 구멍처럼 깜박임.

```
그룹 A ─────────── 그룹 B
  A의 벽 →  |  ← B의 벽
            (같은 좌표에 벽 2개 → Z-fighting)
```

**해결**: `buildGroupCliffsDeduplicated` 구현.
- `buildExtrudedPolygon`을 `includeWalls=false`로 호출 → **상단면만** 생성
- 모든 활성 그룹의 외곽 링 엣지를 수집, 0.5m 정밀도로 해시
- 엣지가 **1개 그룹**에만 존재 → 외곽 경계 → Y=-1.0까지 전체 벽 1개
- 엣지가 **2개 그룹**에 공유 → 내부 경계 → 고도 차이만큼만 벽 1개

```
그룹 A ─────────── 그룹 B
   top면만          top면만
          ↕
    공유 경계에 벽 1개만
   (고도차만큼 생성, Z-fighting 없음)
```

이 원리는 기존 `buildAdjacentCliffs`(파티션 단위)와 동일하며 그룹 단위로 확장한 것.

### 6. 3D 렌더링 파이프라인 전환 (Phase 2/3)

**목표**: 전체 레이어를 "2D 오버레이 방식" → "3D 씬 방식"으로 정리.

#### World Layer (depthWrite=true로 전환)
| 컴포넌트 | 변경 내용 |
|----------|-----------|
| MapTiles | `showElevation ON` 시 `return null` (extruded 메시가 완전 덮음) |
| DongGroundMesh | `depthWrite={true}` |
| ZoneOverlay 바닥 폴리곤 | `showElevation ON` 시 `depthWrite=true` + `AlwaysStencilFunc` |
| SeoulTerrain 도로 | `showElevation ON` 시 `transparent=false`, `depthWrite=true`, `depthTest=true` |
| CityBlockOverlay flat blocks | `depthWrite={!isTransparent}` (불투명 블록만 depth 기록) |

#### UI Overlay Layer (depthTest=false로 전환)
| 컴포넌트 | 변경 내용 |
|----------|-----------|
| SeoulDistrictOverlay | 구/동 경계 포스필드 `depthTest={false}` |
| GroupColorOverlay | 전체 fill/line `depthTest={false}` |
| PartitionBoundaryOverlay | renderOrder 14/15/16 `depthTest={false}` |
| ZoneOverlay ZoneLineMesh | 이미 적용돼 있음 |

## 교훈

1. **Z-fighting은 엣지 중복제거로 해결**: 같은 위치에 메시를 두 번 그리면 반드시 Z-fighting. 공유 경계 엣지는 한 번만 생성해야 한다.
2. **depthWrite=false 남용 금지**: 오버레이가 필요한 UI 레이어에만 `depthTest=false`. 지형/월드 메시는 반드시 `depthWrite=true`.
3. **DoubleSide + depthWrite=true 조합**: extruded 박스 내부에서 top face가 보이려면 DoubleSide 필수.
4. **스텐실 마스크와 elevation 높이 불일치**: DongMask가 Y=0에 있으면 elevated 바닥(Y>0)에서 `EqualStencilFunc` 실패 → `AlwaysStencilFunc`으로 대체.
