## nginx 경고 분석

### 1. `protocol options redefined for 0.0.0.0:443`
- 배포 시 아래 경고가 출력됐다.
  - `/etc/nginx/sites-enabled/p6ix.cloud:29`
  - `/etc/nginx/sites-enabled/sogething:119`
  - `/etc/nginx/sites-enabled/stock.sogething.com:21`
- 원격 서버 설정을 직접 확인한 결과:
  - `p6ix.cloud:29`는 `listen 443 ssl http2;`
  - `stock.sogething.com:21`도 `listen 443 ssl http2;`
  - `sogething:119`는 `listen 443 ssl;`
  - 이번에 배포한 `game.sogething:23`도 `listen 443 ssl;`
- 의미:
  - 같은 `0.0.0.0:443` 소켓에 대해 어떤 서버 블록은 `http2`를 포함하고 어떤 서버 블록은 포함하지 않아 nginx가 프로토콜 옵션이 재정의됐다고 경고한다.
  - 이 경고는 `game.sogething` 단독 문제라기보다 서버 전체에서 443 listen 옵션이 일관되지 않게 섞여 있는 구조 때문에 발생한다.
- 영향:
  - nginx 문법 검사는 통과했고 reload도 성공했다.
  - 즉시 서비스 중단을 일으키는 오류는 아니지만, 운영 설정이 일관되지 않다는 신호다.

### 2. `conflicting server name "sogething.com" on 0.0.0.0:80, ignored`
- 원격 서버의 `/etc/nginx/sites-enabled/sogething` 파일을 확인한 결과, `sogething.com`과 `www.sogething.com`이 포트 80에서 두 번 선언되어 있었다.
- 첫 번째 선언:
  - `/etc/nginx/sites-enabled/sogething:23`
  - `listen 80;`
  - `server_name sogething.com www.sogething.com;`
- 두 번째 선언:
  - `/etc/nginx/sites-enabled/sogething:131`
  - `listen 80;`
  - `server_name sogething.com www.sogething.com;`
- 의미:
  - 동일한 호스트명과 동일한 포트를 가진 서버 블록이 중복 선언되어 뒤쪽 선언 일부가 무시된다.
  - 이 경고는 `game.sogething.com` 설정 때문이 아니라 기존 `sogething` 설정 파일 내부 중복 때문에 발생한다.

### 3. `game.sogething` 설정 자체 상태
- `/etc/nginx/sites-enabled/game.sogething`는 다음 구조로 확인됐다.
  - `listen 80;` + `server_name game.sogething.com www.game.sogething.com;`
  - `listen 443 ssl;` + 동일한 `server_name`
  - `/`는 `localhost:3100` 프록시
  - `/api/`, `/docs`, `/openapi.json`은 `localhost:8100` 프록시
- 현재 `game.sogething.com` 자체에 대해 중복 `server_name` 충돌은 확인되지 않았다.
- 따라서 이번 경고는 신규 사이트 오동작보다는 기존 nginx 운영 설정 누적 문제로 보는 것이 맞다.

## 후속 작업 제안
- `/etc/nginx/sites-available/sogething`에서 포트 80 기준 중복된 `server_name` 블록을 정리한다.
- 서버 내 모든 사이트의 `listen 443` 옵션을 `ssl`만 쓸지 `ssl http2`까지 통일할지 결정하고 일괄 정리한다.
- Node 버전을 `camera-controls` 요구 사항에 맞게 업그레이드할지 검토한다.
- 프론트 번들 chunk 분할 전략을 검토한다.

## 결론
- 배포 실패의 직접 원인은 `server.ps1`의 프로젝트 루트 계산 오류였다.
- 해당 오류 수정 후 배포는 정상 완료됐다.
- nginx 경고는 이번 `game.sogething` 배포 파일 자체의 치명적 문제보다, 기존 서버의 `sogething` 설정 중복과 여러 사이트 간 `listen 443` 옵션 불일치에서 비롯된 것으로 확인됐다.
