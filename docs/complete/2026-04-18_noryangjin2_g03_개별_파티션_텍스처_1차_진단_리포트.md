## 화면 기준 1차 진단

## 1. 좋은 점

### 1. polygon 마스크 정합 자체는 잘 맞는다

각 텍스처가 대체로 파티션 외곽 모양을 따라가고 있다.
즉, 생성 이미지를 polygon 마스크로 잘라 world polygon에 맞게 입히는 후처리 방향은 유효하다.

이건 중요한 성과다.
지금 파이프라인에서 버릴 부분이 아니라 유지해야 하는 부분이다.

### 2. group 공유 1장보다 개별 파티션 구조가 맞다

이전 `group.png` 공유 방식은 그룹 전체가 한 장의 큰 질감을 나눠 쓰는 느낌이 강했다.
반면 지금은 각 파티션마다 개별 이미지가 들어가므로,
구조적으로는 네가 원한 방향에 더 가깝다.

즉:

- 그룹 한 장 공유 방식은 부적합
- 개별 partition 파일 구조는 적합

이 판단은 이번 테스트로 명확해졌다.

---

## 2. 현재 보이는 핵심 문제

### 1. 인접 파티션 간 시각 연속성이 약하다

지금 화면에서는 각 파티션이 "따로 생성된 이미지 조각"처럼 보인다.

구체적으로는:

- 색감이 미묘하게 다름
- 잔디 밀도와 방향성이 다름
- 이끼/흙 패턴이 파티션 경계에서 끊김
- 좁은 길처럼 이어져야 할 면도 한 장면으로 읽히지 않음

즉, 개별 생성은 성공했지만 **연속된 지형처럼 읽히는 데는 아직 실패**한 상태다.

### 2. 경계부에서 이미지 잘림 티가 난다

polygon 마스크 정합은 맞지만,
시각적으로는 "사각형 생성 이미지를 잘라 붙인 흔적"이 일부 보인다.

대표 증상:

- 경계 근처 디테일이 갑자기 끊김
- 잔디 결이 경계에서 단절
- 일부 파티션은 중앙 디테일은 괜찮지만 가장자리로 갈수록 부자연스러움

즉, 마스크는 맞지만 **마스크 이후 가장자리 품질 보정이 더 필요**하다.

### 3. 절벽/측면 UV와 상부 텍스처가 따로 놀아 보인다

스크린샷 기준으로 상부 바닥은 녹지형인데,
측면 절벽 패턴은 별도 타일 패턴으로 반복되어 보인다.

그래서:

- 상부는 자연 지형
- 측면은 기계적인 반복 패턴

처럼 보여 일체감이 떨어진다.

이건 바닥 텍스처 문제만이 아니라,
현재 cliff/wall 쪽 텍스처 처리와도 연결된 문제다.

### 4. gameplay readability 기준으로는 아직 불안하다

지금 결과는 보기에는 이전보다 자연스러울 수 있지만,
게임 플레이용 바닥으로는 아직 불안정하다.

이유:

- 세부 디테일이 파티션마다 제각각이라 화면이 지저분해 보일 수 있음
- 길/빈터/녹지 구분이 플레이 정보로 읽히기보다 텍스처 차이로만 보임
- 확대 시에는 괜찮아 보여도 실제 이동 화면에서는 구역 읽기성이 흔들릴 수 있음

즉, 현재 결과는 "보기 좋은 샘플" 수준이지,
"플레이를 돕는 레벨 바닥"으로는 아직 검증이 필요하다.

## 원인 추정

## 1. 현재 생성은 위성 reference 기반이 아니다

이번 `g03` 개별 파일들은 위성 캡처나 front reference를 입력으로 쓴 결과가 아니다.
기존 `generate_partition_textures.py --per-partition` 방식으로 생성한 prompt 기반 결과다.

그래서 현재 결과는 아래 성격을 가진다.

- 파티션마다 개별 생성
- bbox와 polygon 마스크는 맞음
- 하지만 인접 파티션끼리 shared source가 없음

결과적으로 구조 정합은 있지만, 시각 연속성은 약할 수밖에 없다.

## 2. 개별 파티션 단독 생성이라 shared context가 없다

인접한 파티션들이 같은 도로, 같은 경사, 같은 녹지 흐름을 공유해야 하는데,
현재는 각 파티션이 따로 생성되므로 이어지는 세계로 읽히기 어렵다.

즉, 이번 단계의 한계는 분명하다.

- 단독 생성은 개별성은 준다
- 하지만 연결성은 약하다

## 3. 생성 목표가 아직 "구조 반영형"이 아니라 "분위기형"이다

현재 prompt 방식은 대체로 theme / landuse / persona 기반 무드 생성에 가깝다.
그래서 지역의 분위기는 낼 수 있지만,
실제 지형 구조나 길의 방향성을 안정적으로 반영하진 못한다.

이게 바로 다음 단계에서 위성 reference나 front reference가 필요한 이유다.
