===================================================
AiSogeThing 게임 구현 현황 스냅샷 (2025-02-25)
===================================================

  현재 구현된 것
---------------------------------------------------
1. Three.js 기반 3D 게임 월드 (RpgWorld.jsx)
   - OrbitControls 마우스 드래그로 카메라 각도 자유 조절
   - WASD 이동 시 카메라 각도를 유지하며 평행이동
   - 마우스 휠 줌인/줌아웃 정상 작동

2. 바닥 지도 레이어 구조 (MapTiles.jsx)
   - 1차 레이어: 실물 CartoDB 인터넷 지도 (도로/하천 노출)
   - 2차 레이어: 커스텀 통이미지 맵 구역 (TILE_SIZE = 2000m, GAP = 100m)
   - 버튼 1개 클릭으로 1차 실물 지도 켜기/끄기 토글 가능

3. 통이미지 맵 구역 세팅 (mapConfig.js)
   - 14개 구역 2000m x 2000m 크기로 배치
   - public/images/ 폴더의 판타지풍 이미지 사용
   - 이미지 종횡비 찌그러짐 없이 Crop to Fit 방식으로 정렬

4. 멀티플레이어
   - WebSocket 기반 위치/스킬/채팅 실시간 동기화

5. 게임 오버레이 UI
   - HP/MP/스킬슬롯/미니맵(리플릿 기반 현실 지도)

  현재 미해결 구조적 한계
---------------------------------------------------
- AI 이미지 생성 API는 텍스트->이미지(Text-to-Image)만 지원함
  원본 지도 사진을 넣어서 판타지풍으로 바꾸는
  이미지->이미지(Image-to-Image) 기능이 현재 API에 없음

- CartoDB 인터넷 지도는 줌 레벨에 따라 타일 크기가 변해서
  그 위에 AI 이미지를 1:1 크기로 맞추는 구조가 복잡해짐

  다음 구현 후보들
---------------------------------------------------
A. osmnx 로 서울 도로/하천/산 윤곽 추출 후 Three.js 렌더링
   -> 단색 색깔 구역으로 지형 표현 (강=파랑, 도로=회색, 초원=초록)
   -> 등고선(고도) 데이터 추가하면 실제 지형 굴곡(노멀맵) 표현 가능

B. Stable Diffusion ControlNet (로컬 GPU 필요)
   -> osmnx 추출 윤곽선을 입력으로 판타지 텍스처 자동 변환
   -> GPU RTX 3060+ 있으면 무료 무제한

C. Replicate API or Leonardo.ai API
   -> GPU 없어도 클라우드로 Image-to-Image 자동 변환
   -> 유료이나 장당 약 1~2원 수준
