# 스킬 시스템 고도화 계획

> 작성: 2026-04-12  
> 목표: 마법사 기반 스킬 2개 추가 + 쿨다운 + 퀵슬롯 드래그앤드랍 + 스탯 연동

---

## 현재 상태 분석

### 현재 스킬
| 스킬 | 트리거 | 투사체 | 데미지 | 쿨다운 |
|------|--------|--------|--------|--------|
| `pyramid_punch` | R키 (Player.jsx onAction) | PunchProjectile (프랙탈 분열) | attack + 8 | **없음** |
| `magic_orb` | 자동사냥 useAutoAttack | MagicOrbProjectile (타겟 추적) | attack + 10 | 1200ms (인터벌) |

### 현재 코드 흐름
```
RpgWorld.jsx
  └─ Player.jsx (onAction → addProjectile + sendSkill)
  └─ useAutoAttack.js (setInterval → addProjectile + sendHit)
  
usePlayerSkills.js  ← R키만 감지, 쿨다운 없음
SKILL_POWER dict    ← 단순 power 값만 존재
```

### 문제점
- 스킬 정의가 백/프론트에 흩어져 있음 (SKILL_POWER, 투사체 컴포넌트, useAutoAttack)
- 쿨다운 없음 → 무한 발사
- 스킬 슬롯 개념 없음 → 키 고정
- MP 소모 없음 (MP 스탯은 있는데 안 씀)
- AoE 스킬 불가 → 단일 타겟 구조만 존재

## 추가할 스킬 2종 (마법사)

### ⚡ lightning_bolt — 번개 화살
- **타입**: 직선 고속 관통 투사체
- **발동**: 플레이어 바라보는 방향으로 직선 발사
- **데미지**: attack × 1.2 + 20
- **MP 소모**: 15
- **쿨다운**: 2000ms
- **시각**: 노란색 얇은 실린더, 빠른 속도(30m/s), 2개 몬스터 관통
- **특징**: 타겟 없어도 발사 가능 (방향 기반)

### ❄️ frost_nova — 프로스트 노바
- **타입**: 자기 중심 AoE (반경 8m)
- **발동**: 즉시 발동 (투사체 없음, 자신 주변 폭발)
- **데미지**: attack × 0.9 + 15 (반경 내 전체 몬스터)
- **MP 소모**: 25
- **쿨다운**: 5000ms
- **시각**: 플레이어 중심에서 빙결 링이 퍼지는 파티클 효과
- **특징**: 다수 동시 피격, 백엔드에서 batch_hit 처리

## 아키텍처 설계

### 1. 스킬 카탈로그 — 단일 진실 소스

**프론트**: `front/src/data/skillCatalog.js`
```javascript
export const SKILL_CATALOG = {
  magic_orb:      { icon: '🔮', nameKo: '마법 구체',   mpCost: 10, cooldownMs: 1200, type: 'target',  damage: 10, mult: 1.0, range: 30, desc: '타겟 추적 마법 구체', levelReq: 1 },
  pyramid_punch:  { icon: '👊', nameKo: '피라미드 펀치', mpCost: 5,  cooldownMs: 800,  type: 'spread',  damage: 8,  mult: 0.8, range: 20, desc: '분열하는 펀치 파동',   levelReq: 1 },
  lightning_bolt: { icon: '⚡', nameKo: '번개 화살',    mpCost: 15, cooldownMs: 2000, type: 'linear',  damage: 20, mult: 1.2, range: 50, desc: '직선 관통 번개',       levelReq: 3 },
  frost_nova:     { icon: '❄️', nameKo: '프로스트 노바', mpCost: 25, cooldownMs: 5000, type: 'aoe_self',damage: 15, mult: 0.9, range: 8,  desc: '주변 AoE 빙결 폭발',  levelReq: 5 },
};
```

**백엔드**: `back/player/services/service.py`
```python
SKILL_CATALOG = {
    "magic_orb":      {"power": 10, "mp_cost": 10, "multiplier": 1.0, "type": "target"},
    "pyramid_punch":  {"power": 8,  "mp_cost": 5,  "multiplier": 0.8, "type": "spread"},
    "lightning_bolt": {"power": 20, "mp_cost": 15, "multiplier": 1.2, "type": "linear"},
    "frost_nova":     {"power": 15, "mp_cost": 25, "multiplier": 0.9, "type": "aoe_self"},
}
```

### 2. 퀵슬롯 구조

```
슬롯 1  슬롯 2  슬롯 3  슬롯 4
  Q       E       R       F
[🔮]    [👊]    [⚡]    [❄️]
```

- localStorage 저장 (`skill_hotbar_slots`)
- 기본값: `['magic_orb', 'pyramid_punch', 'lightning_bolt', 'frost_nova']`
- 슬롯당 쿨다운 오버레이 (시계방향 sweep)

### 3. 데미지 계산 (스탯 반영)
```
final_damage = max(1, floor(player.attack × skill.multiplier) + skill.power)
```
장비 강화 시 `player.attack` 값이 올라가므로 자동 반영됨.

### 4. MP 소모
- 스킬 발동 전 MP 확인 → 부족하면 발동 불가 + UI 표시
- 백엔드에서도 MP 검증 (클라이언트 신뢰 금지)
- MP 자동 회복: 초당 2 (게임루프 또는 클라이언트 타이머)

## 파일별 작업 목록

### 신규 파일

#### `front/src/data/skillCatalog.js`
- 스킬 전체 정의 (아이콘, 이름, 쿨다운, 타입, 스탯)
- 프론트 전체에서 import해서 사용

#### `front/src/hooks/useSkillHotbar.js`
- 슬롯 배열 상태 (`slots: [skillId, skillId, skillId, skillId]`)
- 쿨다운 추적 (`cooldowns: { magic_orb: remainingMs, ... }`)
- 슬롯 교체 함수 (`setSlot(slotIdx, skillId)`)
- localStorage 동기화
- MP 소모 처리 (`useSkill(slotIdx)` → MP 차감, 쿨다운 시작, 발동 콜백 호출)

#### `front/src/ui/SkillHotbar.jsx`
- 4슬롯 바 (화면 하단 중앙)
- 각 슬롯: 이모지 아이콘 + 키 바인딩 표시 (Q/E/R/F) + 쿨다운 sweep overlay
- MP 부족 시 슬롯 어둡게 표시
- 드래그 수신 (onDrop → setSlot)

#### `front/src/ui/SkillPanel.jsx`
- 보유 스킬 목록 (현재는 전체 공개, 추후 레벨 제한)
- 각 스킬: 아이콘 + 이름 + 설명 + 쿨다운/MP 정보
- 드래그 소스 (HTML5 drag API)
- 인벤토리 모달과 동일한 스타일 (탭으로 연결 예정)

#### `front/src/entity/player/projectile/LightningBoltProjectile.jsx`
- 빠른 직선 투사체 (speed: 30m/s)
- 노란색 얇은 실린더 메시
- 최대 2개 몬스터 관통 후 소멸
- `sendHit` 로 piercedCount 추적

#### `front/src/entity/player/projectile/FrostNovaEffect.jsx`
- 투사체 없음 — 즉시 AoE 이펙트
- 빙결 링 애니메이션 (scale 0→8m, opacity 1→0, 0.5초)
- 발동 즉시 `sendHit({ skillName: 'frost_nova', aoe: true })`
- 백엔드에서 반경 내 몬스터 전체 피격 처리

### 수정 파일

#### `front/src/entity/player/logic/usePlayerSkills.js`
**현재**: R키 → onTrigger 단순 호출  
**변경**: 4개 키(Q/E/R/F) 각각 → 해당 슬롯 스킬 발동, 쿨다운 체크

```javascript
// 변경 후 구조
const KEY_TO_SLOT = { skill1: 0, skill2: 1, skill3: 2, skill4: 3 };
// actions.skill1~4 를 각각 감지
// cooldownRef[slotIdx]가 0이고 MP 충분하면 발동
```

#### `front/src/hooks/useAutoAttack.js`
**현재**: 타겟 있으면 magic_orb 고정 발사  
**변경**: hotbar 슬롯 1 스킬로 자동 발사 (슬롯 1이 magic_orb가 아닐 수도 있음)  
- `activeSkillId`를 hotbar 슬롯 1에서 가져옴
- 해당 스킬의 쿨다운/타입에 맞게 발사 방식 분기

#### `front/src/entity/world/RpgWorld.jsx`
- `lightning_bolt` 투사체 추가 (LightningBoltProjectile import)
- `frost_nova` AoE 이펙트 렌더링
- `useSkillHotbar` 연결 (playerRef에서 방향 추출하여 스킬 발동)
- `sendHit` 에 `aoe: true, radius: 8` 필드 추가 전달 가능하게

#### `front/src/engine/useGameSocket.js`
- `case 'skill':` 분기에 `lightning_bolt`, `frost_nova` 추가
- `frost_nova`는 투사체 없이 즉시 AoE 이펙트만 재생

#### `front/src/ui/GameOverlay.jsx`
- `SkillHotbar` 컴포넌트 하단에 추가
- `SkillPanel` 토글 버튼 추가 (인벤토리 옆)
- `myStats.mp` 표시 (StatusPanel에 MP 바 추가)

#### `front/src/ui/overlay/StatusPanel.jsx`
- MP 바 추가 (현재 HP 바 아래)

#### `back/player/services/service.py`
- `SKILL_POWER` → `SKILL_CATALOG` (multiplier, mp_cost, type 포함)
- `calc_damage(player_attack, skill_name)` → multiplier 반영
- `handle_frost_nova(user_id, radius, monster_manager, player_manager)` 신규 함수
  - 플레이어 위치 기준 반경 내 몬스터 전체 일괄 피격

#### `back/player/routers/router.py`
- `msg_type == 'hit_monster'` 에 `aoe: true` 처리 분기 추가
- frost_nova 수신 시 `handle_frost_nova()` 호출 → batch 결과 브로드캐스트

#### `back/monster/managers/MonsterManager.py`
- `batch_hit(monster_ids: list[int], damage: int)` 메서드 추가
  - 여러 몬스터를 한 번에 피격 처리
  - 사망 처리 + changed_monsters 반환

## 주의사항

- **WebSocket `hit_monster` 메시지 구조** 변경 시 프론트/백 동시 수정
- **frost_nova는 클라이언트가 반경 계산하지 않음** — 서버에서 플레이어 위치 기준 처리 (보안)
- **기존 R키 pyramid_punch 동작** — Phase 1에서 슬롯 3에 기본 배치, 하위호환 유지
- **자동사냥(useAutoAttack)** — 슬롯 1 스킬이 frost_nova이면 자동사냥 중 AoE 발동 가능 → 밸런스 주의
- **MP 자동 회복** — 백엔드 게임루프에 추가하거나 프론트 타이머로 처리 중 하나 선택 (처음엔 프론트 타이머로 간단히)
