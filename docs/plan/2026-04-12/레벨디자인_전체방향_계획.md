# 레벨 디자인 전체 방향 계획

> 작성: 2026-04-12  
> 목적: 서울 + 판타지 모바일 자동사냥 RPG 레벨 디자인 구조 확정 및 구현 우선순위 정리  
> 기반: agents/planning/game_direction.md, 현재 구현 상태 전체 분석

---

## 게임 정체성 재확인

| 항목 | 방향 |
|------|------|
| 플랫폼 | 오라클 클라우드 멀티 웹, 세로 모바일 |
| 핵심 루프 | 자동사냥 → 도감/펫 수집 → 퀘스트 → 경제(경매장) → 커뮤니티 |
| 테마 | 서울 실제 지도 + 판타지 RPG 감성 |
| 개발 규모 | 1인 개발 → 단순하되 확장 가능한 구조 우선 |
| 참고 유형 | 반자동 파밍 + 도감 수집형 (메이플, 라이즈오브킹덤 혼합) |

---

## 핵심 문제 정의

> **"서울 지도는 있는데 왜 저 동네 가야 하냐"가 없다.**

파티션(동네) 구역과 게임 진행이 연결되지 않은 것이 현재 가장 큰 공백.  
레벨 디자인 = 이 질문의 답을 구역 단위로 채우는 것.

---

## 구조 원칙

### 원칙 1: 동(洞) = 게임 구역

서울 424개 법정동을 게임 구역으로 사용.  
각 동마다 **레벨 범위, 속성 테마, 몬스터 테이블, 퀘스트, 도감 항목** 부여.

### 원칙 2: 레벨 잠금 금지 — 자연 도태 방식

레벨 게이트로 구역을 막지 않는다.  
몬스터 레벨이 높아서 자연스럽게 못 버티는 구조.  
자동 파밍이 구역 적합도를 스스로 판단하게 된다.  
사망 시 시작 구역 리스폰.

### 원칙 3: 루프 완성 우선, 파티션 확장 나중

노량진1~2동만으로 전체 루프(사냥→도감→퀘스트→펫→다음 구역 욕구)를 완성한다.  
그 루프가 완성되면 나머지 동 확장은 **컨텐츠 찍어내기**가 된다.

## 구역 테마 설계 (서울 동별 속성 배분)

| 지역 | 레벨 범위 | 속성 테마 | 배경 근거 |
|------|-----------|-----------|-----------|
| 노량진1동 | 1 ~ 10 | 물(水) | 수산시장, 한강 접경 |
| 노량진2동 | 8 ~ 18 | 지(知) | 고시촌, 학원 밀집 |
| 대방동/신대방 | 15 ~ 25 | 바람(風) | 여의도 인접, 개방 지형 |
| 영등포구 일대 | 20 ~ 35 | 불(火) | 공업/시장 지역 |
| 여의도 | 30 ~ 45 | 금속(金) | 금융 중심지, 섬 지형 |
| 마포/홍대 | 30 ~ 45 | 혼돈(混) | 예술/지하 문화 |
| 강남/서초 | 45 ~ 60 | 번개(電) | 현대/기업 빌딩 |
| 종로/인사동 | 55 ~ 70 | 빛(光) | 고궁, 전통문화 |
| 남산 | 65 ~ 80 | 고대(古) | 서울 중심부, 역사 |
| 한강 수역 | 70+ | 심연(深) | 수계 보스 구역 |
| 특수 던전 | 제한 없음 | 혼합 | 직접 조작 보스 전용 |

> 1차 구현은 노량진1~2동만. 테마 속성은 향후 구역 추가 시 위 표를 기준으로 확장.

## 도감 시스템 (핵심 리텐션)

도감이 곧 장기 플레이 동기다. 단순하되 명확한 수집 목표를 제공한다.

### 몬스터 도감

```
각 동 서식 몬스터 3~5종 배정
처치 누적 횟수 → 도감 자동 등록
  - 1마리: 실루엣 공개
  - 10마리: 이름/설명 공개
  - 50마리: 드롭 아이템 목록 공개
  - 100마리: "정복자" 칭호 + 해당 동 고유 아이템

도감 완성도 % → 구역별 진행률 표시 (도감 화면)
```

### 아이템 도감

```
획득한 아이템 자동 등록 (처음 획득 시 "신규 등록!" 연출)
희귀도별 수집 개수 / 전체 개수 % 표시
전 아이템 도감 완성 시 특별 칭호
```

### 탐험 도감

```
동(洞) 파티션을 N개 이상 방문하면 해당 동 "탐험 완료" 등록
구 단위 전체 완료 시 구 고유 칭호
```

## 펫 시스템

### 획득 방법

```
몬스터 처치 시 낮은 확률(0.1~0.5%)로 "펫 알" 드롭
  - 알 외형 = 해당 몬스터 속성 색상
  - 알 부화 조건: 일정 시간 경과 또는 골드/재료 소비

부화된 펫 = 해당 몬스터 미니 3D 버전
  - 플레이어 주변 공전 (Three.js 오브젝트)
  - 도감 자동 등록
```

### 펫 효과

```
경험치 보너스 펫: 파밍 중 EXP +5~15%
골드 보너스 펫: 드롭 골드 +5~10%
아이템 보너스 펫: 드롭률 +3~8%
탐험 펫: 이동 속도 소폭 증가

펫 강화: 골드 소비 → 효과 +증가 (골드 소비처 역할)
복수 펫: 최대 3마리 동시 장착
```

## 퀘스트 시스템 (단순 3종)

1인 개발이므로 퀘스트 타입을 3개로 제한, DB 기반으로 찍어내기 구조.

### 타입

| 타입 | 예시 | 보상 |
|------|------|------|
| 처치 | "노량진1동 슬라임 30마리 처치" | 골드 + EXP |
| 수집 | "슬라임 젤 10개 모아오기" | 희귀 아이템 |
| 탐험 | "노량진1동 파티션 15개 방문" | 펫 알 |

### NPC 연결

```
동별 대표 NPC 1명 배치 (파티션 centroid 기준)
NPC 클릭 → 퀘스트 수락/완료 대화창
NPC 외형은 해당 동 테마 반영 (수산시장 상인, 고시생, 등)
```

### DB 구조

```sql
quest_template
  id, dong_id, type(kill/collect/explore),
  target_id, target_count,
  reward_gold, reward_exp, reward_item_id, reward_pet_egg_type

character_quest
  id, character_id, quest_id, progress, status(active/done)
```

## 경제 시스템 (골드 소비처 필수)

현재 골드를 쓸 곳이 없어 경제 테스트 불가. 골드 소비처가 없으면 게임 무너진다.

### 골드 소비처 우선순위

```
1. 상점 (HP포션, 소모품) — 가장 먼저
2. 펫 알 부화 / 펫 강화
3. 장비 강화 (+1 ~ +10, 실패 시 골드만 소모)
4. 경매장 (유저 간 거래) — 커뮤니티 핵심
5. 특수 던전 입장권 구매
```

## 구역별 몬스터 배정 구조

### DB 설계

```sql
world_zone
  dong_id         FK → dong 테이블
  level_min       INT
  level_max       INT
  theme           ENUM (water/knowledge/wind/fire/metal/chaos/electric/light/ancient/deep)
  spawn_density   INT  -- 파티션당 동시 스폰 수
  boss_template_id FK → monster_template (nullable)

zone_monster_table
  dong_id
  monster_template_id
  weight          INT  -- 스폰 가중치 (높을수록 자주 등장)
  is_boss         BOOL
```

### 몬스터 스폰 연동

현재 하드코딩 좌표 → 파티션 centroid 기반으로 전환 (A1 작업).  
`zone_monster_table` 기준으로 구역별 몬스터 종류 자동 결정.

## 구현 우선순위 (혼자 개발 관점)

### Phase 1 — 루프 최소 완성 (노량진1~2동)

- [ ] `world_zone` 테이블 생성 + 노량진1~2동 데이터 INSERT
- [ ] `zone_monster_table` 생성 + 몬스터 가중치 배정
- [ ] 몬스터 스폰 파티션 centroid 기반으로 전환 (하드코딩 제거)
- [ ] 몬스터 도감 DB (`monster_codex`) + 처치 횟수 누적 로직
- [ ] 몬스터 도감 UI (도감 화면, 실루엣 → 공개 단계)
- [ ] 상점 UI + 포션 구매 (골드 소비처 1순위)
- [ ] NPC 엔티티 최소 구현 (3D 오브젝트 + 클릭 대화창)
- [ ] 퀘스트 처치/수집 타입 백엔드 + UI

### Phase 2 — 펫 + 경제

- [ ] 펫 알 드롭 로직 (몬스터 처치 시 낮은 확률)
- [ ] 펫 부화 + 3D 미니 오브젝트
- [ ] 펫 도감 UI
- [ ] 장비 강화 시스템
- [ ] 경매장 기반 설계

### Phase 3 — 구역 확장

- [ ] 노량진2동 → 영등포구 → 마포 순 OSM 파티션 완성
- [ ] 구역별 몬스터/퀘스트/NPC 데이터 찍어내기
- [ ] 탐험 도감 완성
- [ ] 보스 몬스터 (구역별 1종)

### Phase 4 — 커뮤니티/길드

- [ ] 길드 생성/가입 (채팅 채널 연동)
- [ ] 구역 점령 (길드전 기반)
- [ ] 성(城) 콘텐츠 — 길드가 특정 구역 보스 처치 시 점령, 세금 수취

---

## 주의사항

- 파티션 100% 완성 기다리지 말 것 — 2개 동으로 루프 먼저 완성
- `world_zone` 테이블 설계는 나중에 바꾸기 가장 어려움 → 처음 잘 잡기
- 골드 소비처 없이 Phase 2 이상 진행 금지 — 경제 붕괴
- 도감은 UI가 게임의 얼굴 → 도감 화면 완성도가 유저 첫인상 좌우
- 퀘스트 타입은 3개로 고정, 복잡한 스토리 퀘스트는 Phase 3 이후
- 성(城) 콘텐츠는 길드 + 경제가 완성된 후 — 건너뛰면 의미 없음
