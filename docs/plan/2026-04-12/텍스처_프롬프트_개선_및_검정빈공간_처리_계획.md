# 파티션 텍스처 프롬프트 개선 및 검정 빈 공간 처리 계획

## 배경 및 목표

DreamShaper XL Lightning으로 생성한 파티션 텍스처 2차 결과물에서 두 가지 문제가 확인됨:
1. **RESIDENTIAL 등 페르소나가 건물 내부/지붕 장면을 생성** → 프롬프트에 건물 관련 단어가 포함되어 있음
2. **polygon 외부가 검정으로 남음** → inpainting은 흰색 마스크 영역만 채움, 외부는 원본 검정 base 그대로

추가 개선 목표:
- Fantasy RPG top-down 느낌 강화
- 파티션 실제 면적(m²) → 프롬프트 scale hint로 반영
- 오브젝트(건물, 소품) 제거, 자연 지형 묘사 극대화
- 페르소나별 프롬프트를 "바닥 지형" 중심으로 전면 재작성

## 분석

### 현재 FLOOR_CONTEXT 문제점

| theme_code | 문제 표현 | 영향 |
|---|---|---|
| `RESIDENTIAL` | "clay tile roofs", "warm rooftop view", "small courtyards" | 지붕 위·마당 내부 장면 생성 |
| `COMMERCIAL` | "colorful awning shadows" | 천막 소품 묘사 유도 |
| `INDUSTRIAL` | "metal grates" | 공장 설비 오브젝트 유도 |
| `RESIDENTIAL_ZONE` | 동일한 RESIDENTIAL와 같은 문제 | 동일 |
| `COMMERCIAL_ZONE` | 동일한 COMMERCIAL과 같은 문제 | 동일 |

### PERSONA_PROMPTS 문제점

현재 persona_prompts는 비교적 잘 작성되어 있으나:
- "korean garden courtyard center" → 마당 장면 일부 유도
- "offerings scattered on ground" → 소품 오브젝트 유도
- `edges_fill` 유도 표현이 없어 검정 경계 표현이 어색

### 검정 빈 공간 해결 방법 비교

| 방법 | 장점 | 단점 |
|---|---|---|
| **A. Python PIL dilate post-processing** | 스크립트 수정만으로 즉시 적용 | 경계가 약간 부드럽지 않을 수 있음 |
| **B. 마스크를 전체 흰색으로 → Three.js polygon clip** | 이미지 전체가 꽉 찬 텍스처 | CityBlockOverlay UV clip 구현 필요 |
| **C. ComfyUI outpainting (마스크 반전)** | 품질 최고 | 워크플로우 수정 복잡 |

**권장: A + B 병행**
- A(post-processing)로 즉시 검정 제거 → 빠른 테스트 가능
- 장기적으로 B(전체 마스크)로 전환하면 품질이 더 좋음

### Phase 2: 검정 빈 공간 채우기 (Python post-processing)

- [ ] `generate_partition_textures.py`에 `fill_black_border()` 함수 추가
  - 방법: PIL `ImageFilter.MaxFilter` 반복 적용 (검정 픽셀을 주변 색으로 팽창)
  - 또는: numpy `ndimage.distance_transform_edt`로 가장 가까운 비검정 픽셀 색 복사
  - 마스크와 결합: polygon 외부 픽셀만 선택적으로 채움

  ```python
  def fill_black_border(img: Image.Image, mask: Image.Image, iterations: int = 20) -> Image.Image:
      """polygon 외부(마스크 검정 영역)를 인접 polygon 내부 픽셀 색으로 채움"""
      result = img.copy()
      for _ in range(iterations):
          expanded = result.filter(ImageFilter.MaxFilter(3))
          # mask 흰색 = polygon 내부(원본 유지), 검정 = 외부(expanded로 채움)
          result = Image.composite(result, expanded, mask.convert("L"))
      return result
  ```

## 주의사항

- `STYLE_PREFIX` 변경 시 village/nature 프리셋(`STYLE_PRESETS`)은 독립적 프롬프트를 가지므로 영향 없음 — 별도 확인 필요
- `fill_black_border()` iterations가 너무 많으면 polygon 경계가 번짐 — 20~40 정도가 적당
- 면적 scale hint는 ComfyUI가 실제로 스케일을 이해하지 못하므로 "small area"/"large area" 수준의 힌트만 의미 있음
- agents 문서(`local_image_generation.md`)를 스크립트와 반드시 동기화해야 함 (소스 오브 트루스)
- 재생성 전 기존 이미지 백업 (`front/public/world_partition/noryangjin2_g04/backup/`)
