# 파티션 폴리곤 마스크 렌더링 정리

## 목적

`world_partition` 또는 `world_partition_group` 텍스처를 생성할 때,
사각형 이미지 전체를 그대로 쓰지 않고 실제 polygon 모양에 맞춰 잘라내는 방식을 기록한다.

이번 문서는 특히 `generate_partition_textures.py`에서 "polygon에 맞게 마스크 써서 렌더링한 방식"을 정리한 참고 문서다.

---

## 적용 위치

- 스크립트: `back/scripts/generate_partition_textures.py`
- 핵심 함수:
  - `compute_bbox(...)`
  - `create_mask(...)`
  - `generate_one(...)`
  - `draw_polygon_outline(...)`

---

## 동작 개요

생성 이미지는 먼저 bbox 기준 직사각형 이미지로 나온다.
그 다음 실제 polygon 경계에 맞춰 후처리로 잘라낸다.

흐름은 아래와 같다.

1. 파티션 또는 그룹 polygon들의 bounding box를 계산한다
2. bbox 기준으로 생성 이미지 크기를 정한다
3. ComfyUI가 먼저 직사각형 이미지를 생성한다
4. 같은 bbox 기준으로 polygon mask를 만든다
5. 생성 이미지에 mask를 씌워 polygon 바깥을 제거한다
6. feather / blur를 적용해 경계가 너무 딱딱하지 않게 만든다
7. 필요하면 outline을 덧그려 정렬을 확인한다

---

## bbox 기준 정규화

핵심은 polygon 좌표를 바로 픽셀로 그리는 게 아니라,
`bbox -> width/height -> normalized pixel space`로 변환한다는 점이다.

이 방식의 장점:

- group / partition 모두 같은 규칙으로 처리 가능
- 생성 해상도가 달라도 같은 정렬 기준 유지
- Three.js 쪽 UV 감각과 맞추기 쉬움

즉 polygon 자체를 직접 렌더하는 게 아니라,
polygon을 포함하는 최소 bbox 안에서 다시 픽셀 좌표로 매핑하는 구조다.

---

## 마스크 생성 방식

`create_mask(...)`는 다음 역할을 한다.

- polygon 좌표를 bbox 기준 픽셀 좌표로 변환
- 흰색 채움 마스크 생성
- polygon 바깥은 검정으로 유지
- feather를 줘서 경계가 너무 톱니처럼 보이지 않게 보정

결과적으로 생성 이미지 위에 아래처럼 쓰인다.

- 흰색 영역 = 남길 부분
- 검정 영역 = 제거할 부분

---

## 실제 클리핑 방식

`generate_one(...)` 안에서는 생성 완료 후 아래 순서로 후처리한다.

1. 생성 PNG 로드
2. `create_mask(...)`로 polygon mask 생성
3. 검정 배경 이미지 생성
4. `Image.composite(...)`로 polygon 내부만 남김
5. `GaussianBlur`를 약하게 한 번 더 넣어 경계 soften
6. 최종 PNG 저장

즉, ComfyUI 내부에서 polygon 모양으로 생성하는 게 아니라,
생성 후 PIL 후처리로 polygon 모양에 맞게 잘라내는 구조다.

---

## 왜 이 방식이 괜찮았는가

이번에 보기에 잘 맞았던 이유는 아래다.

- world polygon 모양 자체는 정확히 보존된다
- bbox 기준 생성이라 스케일 계산이 일정하다
- 경계 feather가 들어가서 너무 딱 끊기지 않는다
- group 모드와 partition 모드에 공통으로 쓸 수 있다

즉, "생성 이미지의 미감"과 "world polygon 정합"을 분리해서 다룰 수 있다는 점이 강점이다.

---

## group 모드와 partition 모드 차이

## group 모드

- group 전체 bbox 기준으로 이미지 1장 생성
- 그 이미지를 group 내부 파티션들이 공유
- 결과 파일 예:
  - `front/public/world_partition/noryangjin2_g03/group.png`

이 경우 시각적으로는 그룹 전체에 한 장을 크게 덮는 느낌이 날 수 있다.

## partition 모드

- 각 `world_partition`마다 개별 bbox 기준으로 이미지 생성
- 각 파티션에 개별 파일 저장
- 결과 파일 예:
  - `front/public/world_partition/noryangjin2_g04/0038.png`
  - `front/public/world_partition/noryangjin2_g04/0039.png`

이 방식이 지금 원하는 구조에 더 가깝다.

---

## outline 확인 방식

정렬 검증이 필요할 때는 `--outline` 또는 `--outline-only`를 쓸 수 있다.

이건 polygon 경계선을 결과 이미지 위에 덧그리는 방식이다.

용도:

- 생성 텍스처가 실제 polygon과 맞는지 확인
- bbox 기준 정렬이 밀렸는지 확인
- 개별 partition 파일의 정합성 검수

---

## 이번 작업에 주는 의미

이번 `g03` 작업에서 먼저 확인된 것은 아래다.

- polygon 마스크 후처리 방식 자체는 유효하다
- group 1장 공유 구조는 생성은 쉬우나, 원하는 `개별 partition 텍스처` 구조와는 다르다
- 다음 단계는 이 마스크 방식을 유지하면서,
  `group 모드`가 아니라 `partition 모드`로 개별 파일을 생성하는 것이다

즉, 바꿔야 하는 건 마스크 방식이 아니라 생성 단위다.
