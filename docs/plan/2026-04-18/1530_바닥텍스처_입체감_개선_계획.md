# 바닥 텍스처 입체감 개선 계획

## 배경 및 목표

ComfyUI로 생성한 이끼낀 돌바닥 텍스처(2D diffuse)가 인게임에서 완전히 평면으로 보임.
동일 이미지를 3D 씬에 적용했을 때 입체감·질감이 살아있도록 개선한다.

현재 파이프라인: `meshBasicMaterial + map` → 조명 없음, 단순 diffuse 출력.

---

## 기법 목록 및 우선순위

### ★★★ 즉각 효과 높음

#### 1. Normal Map (법선 맵)
- **원리**: 각 픽셀에 표면 방향(XYZ 법선)을 저장 → 가짜 요철 생성
- **제작**: ComfyUI 내 `DepthAnything` 또는 `IC-Light`, 또는 무료 툴 `NormalMap-Online` / `Materialize`에 diffuse 넣으면 자동 추출
- **Three.js**: `meshStandardMaterial` 또는 커스텀 셰이더에 `normalMap` 파라미터
- **비고**: meshBasicMaterial은 노멀맵 지원 안 함 → `meshStandardMaterial`로 교체 필요, 또는 직접 셰이더 작성

```glsl
// fragment shader 핵심
vec3 normal = texture2D(uNormalMap, vUv).rgb * 2.0 - 1.0;
float diff = max(dot(normal, lightDir), 0.0);
outColor = diffuseColor * (ambient + diff);
```

#### 2. Ambient Occlusion (AO) 맵
- **원리**: 오목한 부분(돌 사이 틈, 이끼 그늘)을 어둡게 → 깊이감
- **제작**: Materialize 또는 xNormal으로 diffuse → AO 자동 추출
- **적용**: diffuse * AO → 최종 색상에 곱하기
- **비고**: AO만 적용해도 입체감 체감이 크다

#### 3. Parallax Offset Mapping (시차 매핑)
- **원리**: Height Map + UV offset → 시점 변화에 따라 돌 표면이 실제로 튀어나온 것처럼 보임
- **제작**: diffuse grayscale → height map (밝은 돌 = 높음, 어두운 틈 = 낮음)
- **Three.js**: 커스텀 셰이더 필요 (약 30줄), `meshStandardMaterial`의 `displacementMap`은 실제 geometry 변형이라 비용 높음
- **비고**: 아이소메트릭 시점에서 효과 체감 중간. 60도 이상 기울기에서 가장 효과적

```glsl
// parallax 핵심 (fragment)
vec2 parallaxUv(vec2 uv, vec3 viewDir, sampler2D heightMap) {
  float h = texture2D(heightMap, uv).r;
  vec2 offset = viewDir.xy / viewDir.z * h * uParallaxScale;
  return uv - offset;
}
```

### ★★ 효과 중간 / 구현 쉬움

#### 4. Fake Rim Light / Edge Glow (가짜 테두리 조명)
- **원리**: 파티션 폴리곤 가장자리에 밝은 윤곽선 → 높이차 시각화
- **구현**: 셰이더에서 UV 가장자리 거리 계산 후 밝기 강조
- **Three.js**: 기존 CliffShaderMat에 추가 가능

#### 5. Diffuse + Roughness Variation (디테일 변화)
- **원리**: 이끼(어둡고 매트) vs 돌(밝고 반사)를 roughness 맵으로 분리
- **제작**: diffuse의 녹색 채널 추출 → 이끼 영역 마스크
- **적용**: 이끼는 낮은 specularity, 돌은 미세 specular highlight

#### 6. 노이즈 오버레이 (fBm Noise Overlay)
- **원리**: 셰이더에서 월드 좌표 기반 fBm 노이즈를 diffuse에 곱하기 → 카메라 이동 시에도 타일링 패턴 없어보임
- **구현**: 현재 CliffShaderMat의 triplanar + fBm 방식을 바닥 셰이더에도 적용
- **장점**: 제작물 없이 셰이더만으로 즉시 적용

```glsl
float noise = fbm(vWorldPos.xz * 0.05);
outColor.rgb *= 0.85 + noise * 0.3; // 명암 변화
```

#### 7. Micro-Detail Tiling (마이크로 디테일)
- **원리**: 고해상도 소형 노이즈/크랙 텍스처를 큰 텍스처 위에 작은 UV 스케일로 블렌드
- **제작**: 무료 tileable 크랙/모래/이끼 detail 텍스처 (1024x1024)
- **적용**: `mix(bigTex, detailTex, 0.25)` — 근거리에서 픽셀 뭉개짐 방지

---

### ★ 효과 낮거나 비용 높음 (나중에 고려)

#### 8. Screen-Space Ambient Occlusion (SSAO)
- **원리**: GPU가 실시간으로 인접 depth 비교 → 오목한 곳 어둡게
- **Three.js**: `@react-three/postprocessing`의 `SSAO` 컴포넌트
- **비용**: GPU 포스트프로세싱 패스 추가 — 모바일 성능 부담

#### 9. Decal System
- **원리**: 특정 위치에 균열, 혈흔, 발자국 등 동적 오버레이
- **Three.js**: `three-mesh-decal` 라이브러리
- **비고**: 현재 파티션 메시 구조가 복잡해 decal projection 적용 난이도 높음

#### 10. Tessellation + Displacement
- **원리**: geometry 정점을 GPU에서 높이맵에 맞게 실제로 밀어냄
- **비고**: WebGL2는 compute shader 없음 → Three.js에서 vertex shader + height map으로 구현
- **비용**: 파티션 폴리곤 subdivide 필요 → 정점 수 증가

## 셰이더 구조 초안 (GroundShaderMat)

```glsl
// vertex
varying vec2 vUv;
varying vec3 vWorldPos;
varying vec3 vWorldNormal;

void main() {
  vUv = uv;
  vWorldPos = (modelMatrix * vec4(position, 1.0)).xyz;
  vWorldNormal = normalize(mat3(modelMatrix) * normal);
  gl_Position = projectionMatrix * modelViewMatrix * vec4(position, 1.0);
}

// fragment
uniform sampler2D uDiffuse;
uniform sampler2D uNormal;    // optional
uniform sampler2D uAO;        // optional
uniform float uAOStrength;    // 0.0~1.0
uniform vec3 uLightDir;       // normalize(-1, 2, -1) 등

void main() {
  vec4 diff = texture2D(uDiffuse, vUv);

  // AO
  float ao = mix(1.0, texture2D(uAO, vUv).r, uAOStrength);

  // Normal
  vec3 n = texture2D(uNormal, vUv).rgb * 2.0 - 1.0;
  float NdotL = max(dot(n, uLightDir), 0.0);

  // fBm noise (타일링 제거)
  float grain = fbm(vWorldPos.xz * 0.08) * 0.15 + 0.85;

  gl_FragColor = vec4(diff.rgb * ao * (0.6 + NdotL * 0.4) * grain, diff.a);
}
```

## 주의사항

- `meshBasicMaterial`을 커스텀 셰이더로 교체할 때 `depthWrite`, `transparent`, `side` 등 기존 속성을 셰이더 material에도 동일하게 유지해야 한다.
- Normal map은 object-space가 아닌 tangent-space 기준으로 생성해야 Three.js 표준과 맞는다.
- AO 맵 강도는 파라미터(`uAOStrength`)로 조절 가능하게 두어 테마별 적용 강도를 다르게 한다.
- 파티션 바닥 UV는 현재 `TILE_SIZE=100` 기준 — normal/AO 맵도 동일 UV 스케일로 제작한다.
