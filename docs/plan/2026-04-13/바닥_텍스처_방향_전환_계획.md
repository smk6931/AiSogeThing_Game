# 바닥 텍스처 방향 전환 계획

> 작성: 2026-04-13  
> 목적: ComfyUI per-partition 이미지 방식 → world-space UV + theme 기반 타일링으로 전환

---

## 배경 및 목표

### 현재 구조 (2026-04-13 기준)

`CityBlockOverlay.jsx`는 두 경로를 병행한다.

| 경로 | 동작 | 문제 |
|------|------|------|
| **A. per-partition 이미지** | ComfyUI로 PNG 생성 → `front/public/world_partition/{g_short}/{p_short}.png` → `usePartitionTextures` 로드 | 자산 수 = 파티션 수, 모양 바뀌면 재생성, 경계 불연속 |
| **B. theme fallback** | `THEME_TEXTURE_MAP` → `fitUV=false` → world-space UV (100m 타일) | 이미 연속적, 이미지 생성 불필요 |

`DongGroundMesh.jsx`는 맨 아래 캔버스 기반 프로시저럴 잿빛 바닥을 깐다 (`renderOrder=-10`).

### 핵심 결론

- **per-partition 이미지(경로 A)는 메인 레이어로 불필요**하다.
- **theme 기반 world-space UV(경로 B)가 이미 작동**하며 경계 연속성도 해결돼 있다.
- ComfyUI 이미지는 특수 구역 decal로만 남긴다.
- 단계적으로 전환하되 경로 A 코드는 당장 삭제하지 않고 비활성화한다.

---

## 분석

### 현재 `THEME_TEXTURE_MAP` (CityBlockOverlay.jsx:93~101)

```js
const THEME_TEXTURE_MAP = {
  sanctuary_green:     '/ground/forest/...',
  ancient_stone_route: '/ground/grounds/...',
  water:               '/ground/ice/...',
  urban_road:          '/ground/grounds/...',
  residential:         '/ground/grounds/...',
  special:             '/ground/p024_scene_a_test/...',
  default:             '/ground/grounds/...',
};
```

- theme_code → 텍스처 파일 1장
- `buildTerrainBlock` `fitUV=false` → `p.x / TILE_SIZE`, `p.z / TILE_SIZE` (world UV)
- **이미 경계 연속성 달성 완료**

### 문제점

1. theme_code 종류가 너무 적다 (6종).
2. `sanctuary_green`, `residential` 등이 같은 텍스처를 공유해 시각 다양성이 낮다.
3. `DongGroundMesh` 프로시저럴 텍스처가 회색 아스팔트 느낌이라 RPG 분위기와 안 맞는다.
4. `GROUND DIR` 선택은 pool 전체를 교체하는 실험 구조인데, 현재 theme 매핑과 연결이 약하다.

## 전환 방향

### Phase 1 — theme 텍스처 강화 (코드 최소 변경)

목표: per-partition 이미지 없이도 지역별로 뚜렷하게 달라 보이도록

**작업 내용**

- [ ] `THEME_TEXTURE_MAP`에 theme_code별 텍스처를 고품질 파일로 재매핑
  - `sanctuary_green` → 숲/잔디 계열 텍스처
  - `ancient_stone_route` → 석재 계열
  - `water` → 물/얼음 계열
  - `urban_road` → 도로/보도 계열
  - `residential` → 주거 흙/포장 계열
  - 새 theme_code 필요 시 추가 (`market`, `temple`, `ruin` 등)
- [ ] `TILE_SIZE` 100m → theme별 다른 크기 지원 (넓은 forest는 200m, 좁은 dungeon은 30m)
  - `THEME_TILE_SIZE_MAP` 추가

**영향 파일**: `CityBlockOverlay.jsx` 상단 상수 블록만 수정

---

### Phase 2 — per-partition 이미지 경로 비활성화

목표: 경로 A(per-partition 이미지 로드)를 기본 OFF로 설정

**작업 내용**

- [ ] `CityBlockContent`에서 `partitionUrls` 로드 조건에 feature flag 추가
  ```js
  const ENABLE_PER_PARTITION_TEX = false; // 기본 비활성
  ```
- [ ] `usePartitionTextures` 훅이 빈 배열을 받으면 아무것도 안 하게 분기 처리
- [ ] `getGroupGeometries`에서 `hasOwnImage=false`로 강제 → 모든 파티션이 theme fallback 사용
- [ ] 결과: `front/public/world_partition/` 이미지 있어도 무시됨

**영향 파일**: `CityBlockOverlay.jsx` 내 `getGroupGeometries`, `CityBlockContent`

---

### Phase 3 — DongGroundMesh 개선

목표: 맨 아래 베이스 레이어도 RPG 분위기에 맞게

**작업 내용**

- [ ] `DongGroundMesh.jsx` 프로시저럴 텍스처 → 실제 이미지 파일로 교체
  - `ground/base/` 폴더에 잔디/흙 계열 타일링 텍스처 배치
  - `getGroundTexture` → `useTexture('/ground/base/base_ground.jpg')`로 교체
  - `TILE_SIZE = 200m` (베이스이므로 넓게)
- [ ] opacity를 0.6 → 0.85로 올려 베이스 레이어가 더 잘 보이도록

**영향 파일**: `DongGroundMesh.jsx`

---

### Phase 4 — 특수 구역 ComfyUI decal (선택 사항, 나중에)

목표: per-partition 이미지를 특수 연출에만 한정 사용

**조건**: `partition.is_special === true` 또는 `theme_code === 'boss_zone'` 등

- ComfyUI 이미지를 베이스 위에 **반투명 overlay mesh**로만 적용
- polygon 전체 UV fit (현재 경로 A 방식) 유지
- 일반 파티션은 경로 B(world-space UV) 사용

## 주의사항

- `front/public/world_partition/` 폴더 이미지는 삭제하지 않는다 (Phase 4 에서 재활용 가능).
- `generate_partition_textures.py` 스크립트도 삭제하지 않는다.
- `THEME_TEXTURE_MAP`의 파일 경로가 실제로 존재하는지 반드시 확인 후 수정.
- `buildTerrainBlock`의 `fitUV=false` 경로 (world-space UV)는 이미 정상 작동 — 건드리지 않는다.
- `elevation_m` 고도 시스템과 독립적 — 텍스처 전환이 고도에 영향 없음.

---

## 한 줄 결론

> **per-partition 이미지 생성은 이제 메인 레이어에서 빠진다.**  
> theme 기반 world-space UV 타일링이 메인이고, ComfyUI는 특수 구역 전용으로 내려간다.
