# 고도 기반 바닥 텍스처 개선 계획

> 작성: 2026-04-13  
> 전제: per-partition 이미지 생성 방식 폐기 → world-space UV + theme + **elevation_m 활용**

---

## 현재 상태 진단

두 번째 스크린샷(localhost)의 계단식 블록은 이미 `elevation_m`이 적용된 상태다.

### 렌더 파이프라인

```
renderOrder -10  DongGroundMesh      (프로시저럴 회색 베이스)
renderOrder   3  dongBasePlate       (동 경계 gap filler, pool 텍스처)
renderOrder   5  blocks (isWall=false) → theme 텍스처 (world-space UV)
renderOrder   5  blocks (isWall=true)  → /ground/cliff/image.png 또는 slope/image.png
```

### 현재 텍스처 선택 로직

```
partTex (per-partition ComfyUI PNG) 
  → 없으면 themeTexMap[themeCode]     (world-space UV)
  → 없으면 pool tex[hashIdx]          (fallback)
```

**문제**: `themeCode`는 같아도 파티션마다 고도가 크게 다를 수 있다.  
→ 주거 5m 파티션과 주거 80m 파티션이 같은 텍스처를 쓴다.

---

## 개선 가능한 세 가지 축

---

### 축 A — 고도 버킷 기반 텍스처 자동 선택 ★ 우선순위 1

#### 원리

`elevation_m` → 버킷 → 텍스처 다르게 선택

```
low   (< 10m)   → 저지대 잔디/수변 텍스처
mid   (10~40m)  → 기본 잔디/흙 텍스처 (현재)
high  (40m+)    → 바위/산악 텍스처
```

#### 구현 위치

`CityBlockOverlay.jsx:getGroupGeometries` 안에서 파티션별로:

```js
const elevBucket = 
  (partition.elevation_m ?? 0) < 10 ? 'low'
  : (partition.elevation_m ?? 0) < 40 ? 'mid'
  : 'high';

const themeCode = `${partition.group_theme_code || 'default'}_${elevBucket}`;
result.push({ geo, themeCode, ... });
```

#### THEME_TEXTURE_MAP 확장 (예시)

```js
const THEME_TEXTURE_MAP = {
  // mid (현재 기본)
  'sanctuary_green_mid':     '/ground/forest/...',
  'residential_mid':         '/ground/grounds/...',

  // low (저지대)
  'sanctuary_green_low':     '/ground/grounds/image copy.png',   // 습지/수변
  'residential_low':         '/ground/city/image copy.png',

  // high (고지대)
  'sanctuary_green_high':    '/ground/grounds/Lucid_Origin_...3.jpg',  // 바위/산
  'residential_high':        '/ground/cliff/image.png',

  // 버킷 없는 폴백 그대로
  'default':                 '/ground/grounds/Lucid_Origin_...0.jpg',
};
```

**효과**: 노량진 고지대(80m+)는 자동으로 바위 계열, 저지대는 잔디/수변 계열  
**코드 변경량**: `getGroupGeometries` ~5줄 + `THEME_TEXTURE_MAP` 상수 확장  
**리스크**: NONE — fallback 체인이 있어 매핑 없으면 default 텍스처 사용

### 축 B — 등고선 선 geometry (contour line overlay) ★ 우선순위 2

#### 원리

`buildAdjacentCliffs`가 이미 파티션 공유 엣지를 계산한다.  
이 엣지를 재사용해 **파티션 상단 면 위에 얇은 선 geometry** 생성 → 등고선 효과.

```
기존: 공유 엣지 → 수직 면(quad) 생성 (절벽 mesh)
추가: 공유 엣지 중 고도차 > 0 → 수평 얇은 box (등고선 선) 생성
```

#### 구현 방식

```js
// 새 함수: buildContourLines(partitions, effectiveScale)
// edgeMap 순회 → elevYs.length >= 2 → yHigh 위에 얇은 quad 생성

const lineHalfW = 0.3; // 0.3m 폭
// p0 → p1 방향 수직 offset → 양쪽으로 0.3m 벌린 4점 → 2 triangle
// y = yHigh + 0.02 (파티션 상단 바로 위)
```

#### 렌더링

별도 `<mesh>` + `meshBasicMaterial color="#c8a84b" opacity={0.6} transparent`  
→ 금색 등고선 선이 계단 경계에 그려짐

**효과**: 두 번째 스크린샷의 계단 경계가 명확한 등고선처럼 보임  
**코드 변경량**: 새 함수 ~50줄 + `CityBlockContent` 렌더 추가  
**리스크**: LOW — 기존 cliff 로직과 독립

---

### 축 C — 절벽 면 고도별 색 변화 ★ 우선순위 3

#### 원리

현재 절벽 wall은 `cliffTex` 단일 텍스처 + 고정 색상.  
파티션 고도에 따라 절벽 면 색 tint를 다르게 준다.

```
낮은 절벽 (< 20m): 흙빛 갈색 #8a6a44
중간 절벽 (20~50m): 회갈색 #7a6850
높은 절벽 (50m+): 바위 회색 #606060
```

#### 구현 위치

`buildGroupBoundaryCliffs`, `buildAdjacentCliffs`에서 `isWall` block 생성 시  
`elevBucket` 필드 추가 → 렌더 단계에서 color 분기.

```jsx
const wallColor = block.isCliff
  ? (block.elevHigh > 50 ? '#606060' : block.elevHigh > 20 ? '#7a6850' : '#8a6a44')
  : '#8a9a6a';
```

**효과**: 계단 측면이 고도에 따라 다른 색 → 자연스러운 지형감  
**코드 변경량**: `buildAdjacentCliffs`/`buildGroupBoundaryCliffs`에 `elevHigh` 필드 추가 + 렌더 color 로직 5줄  
**리스크**: NONE

## 구현 우선순위 체크리스트

### Phase 1 — 축 A (고도 버킷 텍스처)

- [ ] `getGroupGeometries`에 `elevBucket` 계산 추가 (3줄)
- [ ] `themeCode`를 `${theme}_${elevBucket}` 형태로 변경
- [ ] `THEME_TEXTURE_MAP`에 `_low`, `_mid`, `_high` suffix 키 추가
  - 사용 가능한 텍스처: `ground/forest/*`, `ground/grounds/*`, `ground/city/*`, `ground/cliff/*`
- [ ] `fallback` 처리: suffix 없는 기존 키로 폴백 (`themeTexMap[code] ?? themeTexMap[base] ?? themeTexMap.default`)
- [ ] 브라우저에서 고지대/저지대 파티션 텍스처 다르게 나오는지 확인

### Phase 2 — 축 C (절벽 색 변화)

- [ ] `buildAdjacentCliffs` 반환 object에 `elevHigh` 필드 추가
- [ ] `buildGroupBoundaryCliffs` 동일하게 `elevHigh` 추가
- [ ] `CityBlockContent` 렌더에서 `wallColor` 분기 수정

### Phase 3 — 축 B (등고선 선 geometry)

- [ ] `buildContourLines(partitions, effectiveScale)` 함수 작성
  - `buildAdjacentCliffs`의 edgeMap 로직 재사용
  - 고도차 있는 엣지 → 상단에 얇은 선 quad 생성
- [ ] `showElevation=true`일 때만 등고선 표시
- [ ] `CityBlockContent`에 등고선 mesh 렌더 추가
  - 금색 반투명 (`#c8a84b`, opacity 0.6)
  - `renderOrder=6` (파티션 상단 바로 위)

## 주의사항

- `getGroupGeometries`의 cacheKey에 elevation 합산이 포함돼 있음 → 버킷 로직 추가해도 캐시 무효화 문제 없음
- `THEME_TEXTURE_PATHS`는 중복 제거 후 `useTexture`에 전달하므로 새 텍스처 경로 추가 시 자동으로 포함됨
- `buildAdjacentCliffs`는 partition fallback 경로에서만 호출됨 (그룹 boundary 모드는 `buildGroupBoundaryCliffs` 사용) — 둘 다 수정 필요
- 고도 데이터(`elevation_m`)가 null인 파티션 다수 존재 → `?? 0` null 체크 필수

---

## 한 줄 결론

> `elevation_m`은 이미 DB에 있고 파티션마다 붙어 있다.  
> 코드 10~20줄 추가로 고지대/저지대 텍스처 자동 분기가 가능하다.  
> 등고선 선은 기존 cliff edge 로직을 재사용해 ~50줄로 구현 가능하다.
