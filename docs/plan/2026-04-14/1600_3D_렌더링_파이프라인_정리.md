# Three.js 렌더링 파이프라인 정리 (2D 오버레이 → 3D 씬)

## 배경 및 목표

현재 모든 레이어가 `depthWrite=false` + `renderOrder` 수동 관리 구조(2D 오버레이 방식).
여기에 elevation(고도) 기반 3D 지형(extruded polygon)을 얹으면서:
- 카메라 회전 시 절벽 깜박임
- 인접 파티션 간 렌더 순서 충돌
- 카메라 각도에 따라 절벽/지형이 뚫려 보이는 문제 발생

**목표**: 디아블로/POE처럼 **3D 씬 + 고정 or 자유 등각 카메라** 구조로 전환.
- World 지오메트리 → `depthWrite=true` (GPU depth sorting)
- UI 정보성 오버레이 → `depthTest=false` (항상 화면 위)
- 360도 카메라 회전 시 깜박임 없음

---

## 분석

### 핵심 개념

```
3D 월드 레이어   = 땅·지형·절벽  → depthWrite=true  (GPU가 앞뒤 판단)
UI 오버레이 레이어 = 경계선·색상  → depthTest=false  (항상 화면 위)
```

### 현재 레이어 전체 현황

| renderOrder | 컴포넌트 | depthWrite | 분류 | 전환 방향 |
|---|---|---|---|---|
| -50 | MapTiles | false | 월드 | showElevation ON 시 숨김 |
| -10 | DongGroundMesh | false | 월드 | true로 전환 |
| 0~1 | ZoneOverlay 폴리곤 | false | 월드 | true로 전환 |
| 2 | SeoulTerrain 도로 (elev ON) | false | 월드 | true + depthTest=true |
| 5 | 파티션 extruded 메시 | **true ✓** | 월드 | 완료 |
| 10~11 | 구/동 경계선 | false | UI | depthTest=false |
| 20~23 | GroupColorOverlay | false | UI | depthTest=false |
| 101~112 | ZoneLine 도로선, 도로 메시 | false | UI/월드 | depthTest=false |

### 영향 범위

- `CityBlockOverlay.jsx` — flat 블록 depthWrite 전환
- `MapTiles.jsx` — showElevation 조건부 숨김
- `DongGroundMesh.jsx` — depthWrite=true
- `ZoneOverlay.jsx` — 폴리곤 depthWrite=true, 라인 depthTest=false
- `SeoulTerrain.jsx` — 도로 depthWrite=true
- `SeoulDistrictOverlay.jsx` — 경계선 depthTest=false
- `GroupColorOverlay.jsx` — depthTest=false
- `PartitionBoundaryOverlay.jsx` — depthTest=false

## 주의사항

1. **투명 머티리얼**: `transparent=true`인 머티리얼은 Three.js가 자동으로 depth 기록 안 함.
   GroupColorOverlay (opacity 0.28~0.36)는 `depthTest=false` + 높은 renderOrder 유지.

2. **showElevation 조건 분기 유지**: OFF 시 기존 2D 오버레이 구조 그대로.
   Phase 2~3 변경은 `showElevation` prop 받는 컴포넌트에만 조건부 적용.

3. **Phase 순서 중요**: 월드 레이어 먼저 고치고(Phase 2) UI 오버레이 분리(Phase 3).
   반대로 하면 UI가 지형 뒤로 사라지는 문제 생길 수 있음.

4. **MapTiles 숨김**: showElevation ON 시 지도 타일이 보이면 extruded 메시 아래로
   depth 충돌 가능성. 완전히 숨기는 게 가장 안전.
