# Title: Multiplayer Optimization Plan
Description: 실시간 이동, 스킬, 몬스터, 지역 조회, 프론트 로딩 최적화를 단계별로 진행하기 위한 작업 문서.
When-To-Read: 멀티플레이 성능 최적화, websocket 부하 감소, AOI 전송, 프론트 번들 분할 작업 시
Keywords: multiplayer, websocket, optimization, aoi, monster, region, polling, lazy-loading, bundle
Priority: high

## 목표

- 플레이어 이동과 스킬 이벤트를 필요한 범위에만 전송한다.
- 몬스터 동기화를 전체 스냅샷에서 delta 기반으로 줄인다.
- 현재 지역 조회와 오버레이 polling 중복을 제거한다.
- 프론트 초기 로딩 비용과 대형 청크를 줄인다.
- 이후 사람 수와 몬스터 수가 늘어도 확장 가능한 구조를 만든다.

## 작업 순서

1. 이동 전송 경로 정리
- 로컬 렌더링용 이동과 서버 동기화용 이동을 분리한다.
- 불필요한 중복 위치 전송을 줄인다.

2. 서버 브로드캐스트 구조 개선
- 전체 순차 전송을 병렬 전송으로 바꾼다.
- 느린 소켓이 전체 전송을 막지 않게 정리한다.

3. 현재 지역 조회 통합
- 현재 위치 기반 지역 조회를 공용 훅으로 모은다.
- 중복 polling API 호출을 제거한다.

4. AOI 전송 적용
- 플레이어 이동, 스킬, 전투 이벤트를 근접 유저 기준으로만 보낸다.
- 본인 시야 기준 몬스터만 다시 동기화한다.

5. 몬스터 delta 동기화
- 전체 몬스터 스냅샷 대신 upsert/remove delta를 사용한다.
- 플레이어별 가시 범위를 기준으로 delta를 분배한다.

6. 프론트 로딩 최적화
- 선택형 패널과 모달은 lazy loading으로 분리한다.
- vendor/manual chunk 분할로 초기 청크를 줄인다.

7. 후속 검증
- 이동, 채팅, 스킬, 몬스터 타격, 지역 표시, 월드 오버레이 동작을 확인한다.
- 남은 병목을 다음 단계 작업으로 정리한다.

## 후속 확장 포인트

- 플레이어 가시 범위를 맵 셀 또는 partition 단위 AOI로 더 세분화
- 몬스터 AI tick도 플레이어 인접 구간 중심으로 부분 실행
- 스킬 이펙트와 투사체도 delta/AOI 이벤트로 통합
- 월드 geometry 생성 결과를 캐시하거나 background chunk로 분리
- 프론트 대형 라이브러리와 관리용 패널을 더 세밀하게 코드 스플릿

## 진행 기록

- 완료: 위치 중복 전송 방지와 서버 전송 경로 정리
- 완료: 서버 브로드캐스트 병렬화
- 완료: 현재 지역 조회 공용화와 중복 polling 제거
- 완료: AOI 기반 플레이어 이동/스킬/전투 이벤트 전송
- 완료: 몬스터 delta 동기화 적용
- 완료: 몬스터 delta의 플레이어별 가시 범위 분배
- 완료: 프론트 lazy loading과 manual chunk 분할 적용
