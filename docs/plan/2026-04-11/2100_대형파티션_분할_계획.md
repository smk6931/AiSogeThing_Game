## 배경 및 목표

현재 노량진1동 공원/숲 파티션 일부가 비정상적으로 크다.  
평균 파티션 면적 8,560m²인데 최대 286,721m²(~30배)짜리가 존재.  
한 파티션 = 한 텍스처이므로 대형 공원이 단조롭게 렌더링됨.  
임계값 이상의 파티션을 Shapely로 자동 분할하여 텍스처 다양성 확보.

**분할 기준**: `area_m2 > 15,000m²` → 분할 대상 (평균의 ~1.75배)

### 분할 방식 옵션

| 방식 | 설명 | 장점 | 단점 |
|---|---|---|---|
| **격자 분할 (Grid)** | bbox를 NxN 격자로 나눈 후 원본 polygon과 교차 | 구현 단순, 균등 크기 | 직선 경계, 자연스럽지 않음 |
| **Voronoi 분할** | 내부 랜덤 시드로 Voronoi 다이어그램 생성 | 자연스러운 경계 | 구현 복잡 |
| **스트립 분할 (Strip)** | 긴 축 방향으로 균등 슬라이스 | 빠름 | 단방향 분할만 |
| **재귀 2분할** | 긴 축 기준 절반씩 분할 반복 | 균형 잡힘 | 직선 경계 |

**권장: 격자 분할 (Grid)**  
target_area 기준으로 목표 분할 수 = `ceil(area_m2 / TARGET_AREA_M2)` 계산.  
각 서브 polygon에 원본 partition의 theme_code, dominant_landuse, admin_area_id 복사.

### 영향 범위

| 테이블 | 변경 | 비고 |
|---|---|---|
| `world_partition` | 대형 파티션 삭제 → 분할된 sub-파티션 INSERT | partition_seq 새로 부여 |

## 파라미터 설계

```python
SPLIT_THRESHOLD_M2 = 15_000   # 이 면적 초과 시 분할
TARGET_AREA_M2     = 8_000    # 목표 서브 파티션 면적 (평균과 유사)
# 분할 수 = ceil(area_m2 / TARGET_AREA_M2)
# 예: 286,721m² → ceil(286721/8000) = 36개로 분할
# 예: 147,649m² → ceil(147649/8000) = 19개로 분할
```

## 주의사항

- `world_level_partition`은 VIEW — 직접 수정 불가, world_partition만 건드림
- 분할 시 `partition_key`는 원본 key + `_s{N}` suffix로 고유성 보장
- `source_feature` JSON은 원본 복사 (OSM 원본 참조 유지)
- 분할 후 반드시 `regroup_partition_members.py` 재실행 필요
- 노량진2동도 동일 스크립트로 적용 가능 (`--dong-osm-id 3879477`)
