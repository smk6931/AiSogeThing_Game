# 파티션 계층 구조 재설계 계획

**작성일**: 2026-04-11  
**상태**: WAITING FOR CONFIRMATION

---

## 배경 및 목표

스크린샷에서 바닥 텍스처가 대부분 어둡고 일부만 렌더링되는 문제.
서울시 행정구역 계층 (시→구→동→통/반) 방식으로 파티션 계층을 재구성하는 것이 유효한지 분석하고 DB 조정 방향 결정.

---

## 분석

### 현재 구조

```
world_admin_area (dong)     = 노량진1동 (1370×1756m)
world_partition_group (21개) = 서브구역 (avg 200-400m radius, avg 9.7개 파티션)
world_partition (222개)      = 개별 필지 (avg ~100m radius, avg 8,560m²)
```

### 스크린샷 문제 원인

| 증상 | 원인 |
|---|---|
| 대부분 어두운 지형 | CityBlockOverlay가 현재 위치 주변 그룹만 렌더링 (인접 700m) |
| 일부 텍스처만 보임 | showCurrentGroupTexture 미활성화 + 텍스처 파일 부족 |
| 쪼개진 느낌 | 배경 레이어 없이 block 레이어만 존재 |

### 서울 행정구역 계층 매핑

```
서울특별시 (시)           → world_admin_area (level=city)   [기존]
└── 자치구 (25개, ~2.7km)  → world_admin_area (level=gu)    [기존]
    └── 법정동 (424개, ~670m) → world_admin_area (level=dong) [기존]
        └── 대구역 (통 수준, ~600m) → world_partition_group (group_level='dong_sub') [신규]
            └── 소구역 (반 수준, ~200-400m) → world_partition_group (group_level='block') [기존 21개]
                └── 필지 → world_partition [기존]
```

## 세 가지 옵션

### Option A. 현재 구조 유지 + 렌더링 수정 (최소 변경)
- 인접 그룹 탐지 반경 700m → 1500m (동 전체)
- showCurrentGroupTexture 기본값 ON
- **장점**: 즉시 적용 가능  
- **단점**: 배경 텍스처 없음, 근본 구조 미해결

### Option B. group_level 추가 (권장, 점진적 계층화)
- `world_partition_group.group_level` 컬럼 추가
- 기존 21개 → `block` 레벨
- 신규 5-6개 `dong_sub` 그룹 → 배경 텍스처 레이어
- **장점**: 최소 DB 변경 + 전체 맵 텍스처 커버 가능
- **단점**: dong_sub 설계 필요, 이중 레이어 렌더링 추가

### Option C. 서울 행정구역 완전 계층화
- world_admin_area에 통/반 레벨 추가
- 완전한 4계층 구조
- **장점**: 가장 확장성 높음
- **단점**: DB 대규모 재설계, 구현 복잡도 최고

## 주의사항

- `world_partition` 테이블은 수정하지 않음 (실제 테이블)
- `world_level_partition`은 VIEW — 직접 UPDATE 불가
- `group_level` 컬럼 추가 마이그레이션은 nullable/default 방식으로 무중단
- dong_sub 그룹은 block 그룹보다 먼저 렌더링되어야 함 (z-index/renderOrder 주의)
- 텍스처 파일 없으면 fallback 색상으로 처리 (기존 theme_code 기반 색상)
