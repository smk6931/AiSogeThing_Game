================================================================================
텍스처 스케일 불균형 문제 해결 계획서 (Texture Scaling Fix Plan)
================================================================================

1. 문제점 분석 (Problem Analysis)
   현재 땅바닥(CityBlockOverlay) 및 동 단위 지면(DongGroundMesh)의 텍스처는
   각 메쉬의 경계 상자(Bounding Box)를 기준으로 0~1 사이로 정규화된 UV를 사용함.
   이로 인해 면적이 넓은 구역은 텍스처가 늘어나고, 좁은 구역은 뭉지는 현상이 발생.
   캐릭터 크기에 대비해 텍스처 밀도가 불규칙하여 월드 몰입감을 저해함.

2. 해결 방안: 월드 좌표 기반 UV 매핑 (World-Space UV Mapping)
   정규화된 UV 대신 실제 게임 좌표(미터 단위)를 기반으로 UV를 생성함.
   - UV.x = p.x / TILE_SIZE
   - UV.y = p.z / TILE_SIZE
   여기서 TILE_SIZE는 텍스처 한 칸이 차지할 실제 미터 거리(예: 10m 또는 20m)임.
   이렇게 하면 모든 구역에서 텍스처가 동일한 비율로 출력되며,
   서로 인접한 블록끼리도 텍스처가 끊김 없이(Seamless) 이어지게 됨.

3. 상세 수정 사항 (Implementation Steps)

   A. CityBlockOverlay.jsx 수정
      - 텍스처 래핑(Wrapping) 방식을 ClampToEdge에서 RepeatWrapping으로 변경.
      - buildTerrainBlock 함수 내 UV 계산식을 월드 좌표 기반으로 변경.
      - 기본 타일 크기(TILE_SIZE) 설정 (약 20m 권장).

   B. DongGroundMesh.jsx 수정
      - 현재 AABB 정규화 방식을 제거하고 월드 좌표 기반 UV로 통일.
      - 절차적 잔디 텍스처의 반복(Repeat) 계수를 새로운 UV 체계에 맞춰 조정.

4. 기대 효과 (Expected Results)
   - 캐릭터가 움직일 때 지면의 텍스처 질감이 일정한 크기로 유지됨.
   - 대규모 면적에서도 텍스처가 늘어지지 않고 고른 밀도를 보여줌.
   - 월드 전체의 시각적 통일성 확보.
