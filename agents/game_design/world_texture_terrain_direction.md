# Title: World Texture Terrain Direction
Description: 파티션 바닥 텍스처, 도로 텍스처, 노이즈 블렌딩, 고도 적용 방향을 현재 구현 기준으로 정리한 작업 문서.
When-To-Read: 월드 바닥 텍스처 시스템 개선, 도로 블렌딩, 지형 고도화, 파티션 기반 월드 표현 방식을 정리하거나 리팩터링할 때
Keywords: world, terrain, ground, road, texture, partition, blend, noise, heightmap, uv, decal
Priority: high

## 목표

- 현재 `GROUND DIR`, `ROAD DIR` 폴더 선택형 실험 구조를 유지하면서도 최종 품질 방향을 명확히 한다.
- 파티션 경계가 보이는 문제와 도로가 스티커처럼 뜨는 문제의 원인을 분리한다.
- "폴리곤마다 이미지 한 장" 방식이 최종 해법인지 여부를 정리한다.
- 다음 구현 우선순위를 문서로 남긴다.

## 현재 상태 요약

현재 구현은 아래 두 축으로 동작한다.

1. 바닥/파티션 계열
- `front/public/ground/<folder>` 구조로 텍스처 세트를 관리한다.
- `GROUND DIR` 선택값으로 현재 사용할 바닥 텍스처 폴더를 바꾼다.
- 파티션/블록 계열 바닥은 `CityBlockOverlay`에서 폴리곤 geometry 위에 텍스처를 입힌다.

2. 도로 계열
- `front/public/road/<folder>` 구조로 텍스처 세트를 관리한다.
- `ROAD DIR` 선택값으로 현재 사용할 도로 텍스처 폴더를 바꾼다.
- 도로는 `SeoulTerrain`에서 선형 geometry를 띠 형태로 만들어 텍스처를 입힌다.

이 구조는 "실험과 교체 속도"에는 좋다. 하지만 최종 월드 표현으로는 한계가 있다.

## 현재 문제

### 1. 파티션 경계가 보임

- 각 파티션 폴리곤이 자기 geometry 안에서 별도 UV를 가진다.
- 그 결과 텍스처 결 방향과 스케일이 폴리곤마다 따로 잡혀 보인다.
- 인접 파티션끼리 동일한 재질 계열이어도 결이 이어지지 않아 경계선이 눈에 띈다.

### 2. 도로가 자연스럽지 않음

- 지금 도로는 폭이 있는 띠 mesh를 별도로 올리는 방식이다.
- 도로 가장자리에 침식, 흙 번짐, 잔디 침범, 알파 페이드 같은 블렌딩이 적다.
- 결과적으로 "지형 위에 도로 스티커를 얹은 느낌"이 강하다.

### 3. 고도감이 부족함

- 현재 바닥은 거의 평면 기반 표현이다.
- 숲, 흙, 물, 도로 모두 수직 고도 차나 미세 노말 변화가 부족하다.
- 그래서 위에서 보면 그림처럼 보이지만, 공간감은 약하다.

## 핵심 판단

### A. 도로 노이즈 블렌딩은 현재 구조에서도 비교적 쉽게 가능하다

이건 맞다.

가능한 이유:
- 도로는 이미 `SeoulTerrain`에서 별도 geometry와 material 경로가 있다.
- 여기서 edge noise, alpha fade, grunge mask, color variation을 추가하기 쉽다.
- 즉 기존 strip road 구조를 유지한 채로도 품질 상승 여지가 크다.

추천 방식:
- 도로 텍스처 자체에 가장자리 마모를 넣는다.
- `uvRepeat`와 함께 noise 기반 밝기/색 변형을 조금 준다.
- 도로 가장자리용 얇은 secondary mesh나 alpha strip을 추가한다.
- `major`, `mid`, `alley`별로 거칠기와 가장자리 침식 정도를 다르게 준다.

결론:
- 도로 쪽은 "구조를 완전히 갈아엎지 않고도" 먼저 개선해볼 가치가 높다.

### B. 파티션 텍스처는 폴리곤 기반 이미지를 다시 생성하는 것이 정답은 아니다

중요하다.

질문:
"파티션 텍스처를 이미지 생성을 폴리곤 기반으로 다시 뽑아야 하냐?"

답:
- 반드시 그럴 필요는 없다.
- 오히려 최종 방향으로는 권장되지 않는다.

이유:
- 폴리곤별 이미지를 따로 만들면 자산 수가 급증한다.
- 폴리곤이 바뀌거나 시드가 바뀌면 이미지 재생성이 필요하다.
- 인접 폴리곤 간 연속성이 다시 깨질 가능성이 높다.
- 지형 전체 일관성보다 "조각난 타일" 느낌이 강해진다.

즉:
- 폴리곤마다 전용 이미지를 한 장씩 만드는 방식은 관리 비용이 크고 확장성이 낮다.

## 더 좋은 방향

### 1. 연속 지형을 기본으로 두기

최종 베이스는 `continuous terrain`이어야 한다.

의미:
- 월드 전체가 하나의 연속된 지형처럼 보여야 한다.
- 텍스처는 파티션 경계와 무관하게 world-space UV 또는 large-scale sampling으로 이어져야 한다.

추천:
- 바닥 베이스는 world-space UV 기반 샘플링
- grass, dirt, rock, moss 등을 노이즈로 섞기
- 파티션은 "재질 가중치"만 제공

예:
- 파티션 A: grass 0.7, dirt 0.2, rock 0.1
- 파티션 B: grass 0.2, dirt 0.6, rock 0.2

이렇게 하면 경계가 "재질 전환 구간"으로 약해지고, 조각난 느낌이 줄어든다.

### 2. 파티션은 텍스처 본체가 아니라 마스크/테마 정보로 쓰기

파티션의 더 좋은 역할:
- theme_code
- landuse_code
- moisture/dryness
- road proximity
- forest density

즉 파티션은 "어떤 재질을 얼마나 섞을지"를 알려주는 데이터 소스가 되어야 한다.

### 3. 폴리곤 기반 생성이 필요하다면 "마스크" 쪽으로 한정하기

폴리곤 기반 생성이 완전히 불필요한 것은 아니다.

유효한 경우:
- 특정 그룹/파티션을 강조하는 특별한 룬 문양
- 이벤트 지역의 고유 지면 문양
- 지역 보스방, 신전, 오염지대 같은 특수 연출

이 경우에도 권장 방식은:
- 폴리곤 전체 텍스처 한 장을 만드는 것보다
- polygon mask + overlay decal 조합으로 가는 것

즉:
- 기본 지형은 연속
- 특수 문양만 polygon overlay

## 구현 우선순위

### 1차
- 도로 edge noise/blend 추가
- 도로 텍스처 폴더별 교체 실험
- 도로 색/거칠기 variation 추가

### 2차
- 파티션 바닥 UV를 world-space 기반으로 바꿔 경계 약화
- block/partition texture를 "정사각 이미지 박기"에서 점진적으로 축소

### 3차
- base terrain layer 도입
- grass/dirt/rock/moss 멀티 레이어 블렌딩
- partition은 재질 weight source로 전환

### 4차
- heightmap 또는 procedural displacement 도입
- 도로가 지형을 따라가도록 terrain conform 적용
- 물/숲/경사에 따른 재질 보정

## 실무 결론

- `GROUND DIR`, `ROAD DIR` 폴더 선택형 실험 구조는 유지할 가치가 있다.
- 도로 노이즈 블렌딩은 지금 바로 시작해도 된다.
- 파티션 텍스처는 "폴리곤별 이미지 재생성"으로 가기보다
  연속 지형 + 파티션 마스크 방향으로 가는 게 더 낫다.
- 즉 현재 방식은 실험용으로는 맞지만, 최종 월드 렌더 구조로는 중간 단계다.

## 한 줄 결론

- 도로: 현재 구조에서 노이즈 블렌딩 개선 먼저 한다.
- 파티션: 폴리곤별 텍스처 이미지 재생성보다, 연속 지형 위에 파티션 마스크를 얹는 방향이 더 좋다.
