## 핵심 가설

> **"서울의 지형은 고정이다. 그 위에 올라가는 모든 것은 플레이어가 만든다."**

현실 서울에서:
- 한강, 남산, 북한산의 지형은 인간이 바꾸지 않는다
- 계단, 다리, 길, 건물은 수백년에 걸쳐 사람이 쌓아왔다

이 게임은 그 과정을 압축 재현한다.

---

## 게임 정체성이 완성되는 이유

### 기존 서울 지형 RPG와 다른 점

| 항목 | 일반 서울 배경 게임 | 이 게임 |
|------|------|------|
| 월드 구조 | 개발자가 수동 배치 | OSM 실측 데이터 기반 |
| 오브젝트 | 개발자가 심어둔 것 | 플레이어가 배치한 것 |
| 내 흔적 | 세션 종료 시 사라짐 | 서버 DB에 영구 보존 |
| 다른 플레이어와 관계 | 같이 싸우는 적/동료 | 내가 만든 길을 걷는 사람들 |

### 사회적 기여 루프

```
몬스터 처치 → 자원 드롭
→ 자원 소비 → 계단/다리/건물 배치
→ 다른 플레이어가 사용 → use_count 누적
→ "더블유의 돌길 — 통행 1,847명"
→ 명성 + 지도에 이름 표시
→ 더 좋은 위치에 더 많이 짓고 싶은 욕구
```

이 루프는 **자동사냥 → 도감 → 경매장** 루프와 병렬로 돌아간다.

## 검증 조건

이 정체성을 확정하려면 아래 조건이 충족되어야 한다:

- [ ] Phase 1 루프 (사냥 → 상점 → 도감 → 퀘스트) 완성
- [ ] DAU 50명 이상 환경에서 테스트
- [ ] 실제로 플레이어가 계단/다리를 배치하는 행동이 자발적으로 발생
- [ ] use_count가 의미 있는 수준(100+)으로 누적되는 오브젝트 존재 확인

## 1인 개발 관점에서의 장점

- 424개 법정동 오브젝트를 개발자가 직접 배치하는 건 불가능
