# Title: Game Direction
Description: 핵앤슬래시 액션 vs 커뮤니티 중심 반자동 파밍 방향 결정 및 근거

Keywords: 게임방향, 기획, 전투, 커뮤니티, 파밍, 자동사냥, 보스, 경매장, 제작, 강화

## 결정된 방향: 세로 (커뮤니티 + 반자동 파밍)

2026-03-29 결정.

### 핵심 컨셉

- 평소 파밍/사냥은 **반자동 또는 자동**으로 처리
- 유저 간 **채팅, 정보 교환, 협력**을 주 콘텐츠로
- **경매장, 제작, 강화, 조합** 등 경제 시스템 적극 구성
- **특수 던전 / 보스**만 직접 조작 (컨트롤 요소 유지)
- 퀘스트는 혼자보다 사람들과 함께 풀어가는 구조

### 선택 이유

- 서울 실제 지도 + 동네 기반 파티션 구조가 이 방향의 뼈대와 맞음
- 노량진/강남/마포 같은 지역 정체성을 커뮤니티로 엮으면 차별화 가능
- 소규모 개발 환경에서 액션 퀄리티 경쟁(Diablo/PoE 수준)은 비현실적
- 커뮤니티 콘텐츠는 유저가 자생적으로 콘텐츠를 만들어냄 → 체류시간 증가

### 기각된 방향: 가로 (액션 핵슬)

- 스킬 이펙트, 히트박스, 보스 패턴 AI 등 개발 비용이 극단적으로 큼
- 콘텐츠 소모 속도 빠름 → 유저 이탈 빠름
- 1인/소규모 개발로는 경쟁력 확보 어려움

### 타협점 (두 방향 혼합)

| 상황 | 방식 |
|------|------|
| 일반 사냥 / 파밍 | 반자동 or 자동 |
| 퀘스트 / 정보 교환 | 채팅 + 커뮤니티 |
| 경제 (거래/제작/강화) | 경매장, 조합 시스템 |
| 특수 던전 / 보스 | 직접 조작 (컨트롤) |

### 앞으로 기획 시 체크 포인트

- 새 콘텐츠가 "혼자 할 수 있나?" → 커뮤니티 연계 방향으로 설계할 것
- 전투 시스템 추가 시 → 자동화 가능한지 먼저 확인
- 보스 난이도 설계 → 특수 던전에만 집중, 일반 필드 보스는 가볍게
- UI 설계 → 채팅/정보 교환 UX 우선순위 높게
