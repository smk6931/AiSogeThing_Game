# Title: 프론트 UI 규칙
Description: 모바일 우선 UI, 레이어 패널, 게임 오버레이 배치와 표현 규칙.
When-To-Read: 프론트 UI 수정, 오버레이 조정, 모바일 대응, HUD 개선 작업 전.
Keywords: frontend, ui, mobile, overlay, hud, minimap, panel, responsive
Priority: high

## 기본 원칙

- 모바일 기준으로 먼저 읽히게 만든다.
- 데스크톱은 정보를 더 많이 보여줄 수 있지만 모바일 배치를 깨면 안 된다.
- 게임 HUD는 지도/월드 레이어보다 우선 읽혀야 한다.

## 배치 원칙

- 상태창은 좌상단에 작고 안정적으로 유지한다.
- 미니맵과 지역 카드류는 우상단 축으로 모은다.
- 전투 버튼, 이동속도, 채팅 입력은 서로 클릭을 가리지 않게 분리한다.

## 레이어 패널 원칙

- 개발용 토글과 플레이용 토글을 섞되, 이름은 짧게 유지한다.
- `경계`, `그룹선`, `미세선`, `현재강조`처럼 기능 기준으로 분리한다.
- 상세 정보는 기본 접기, 필요 시 펼치기 방식으로 간다.

## 월드 표현 원칙

- 기본 바닥과 파티션 바닥이 동시에 강하게 겹치지 않게 한다.
- 디버그 레이어는 플레이 레이어보다 시각적으로 뒤에 두거나 더 연하게 둔다.
