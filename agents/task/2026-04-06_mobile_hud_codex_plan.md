## 목적

- 모바일 화면에서 상단 HUD 겹침과 우하단 전투 UI 배치를 정리한다.
- 자동사냥 버튼을 현재 좌상단 스탯 패널 기준에서 분리하고 우하단 전투 조작 영역으로 이동한다.
- 이동 스틱과 Burst 버튼의 좌우 기준선을 맞춰 모바일 조작 체감을 정리한다.
- 미니맵 하단 메뉴와 사이드 메뉴의 역할을 다시 정리한다.
- 도감(`GameCodex`)에 아이템 도감 연동을 붙일 수 있도록 별도 Phase로 분리 관리한다.

---

## 현재 확인된 이슈 (2026-04-06 기준)

### 모바일 HUD / 입력

- 상단 좌측 스탯 패널과 상단 툴 버튼 줄이 화면 폭이 좁을 때 답답하게 보인다.
- 자동사냥 토글이 좌상단 스탯 패널 내부에 있어 모바일 전투 조작 흐름과 맞지 않는다.
- 이동 스틱과 우하단 Burst 버튼의 시각적 좌우 정렬이 어긋나 보인다.
- 미니맵 아래 메뉴 버튼과 우측 사이드 메뉴의 기능 경계가 모호하다.
- 모바일 줌 UI는 커졌지만 상단 툴군 전체 재배치가 아직 끝난 상태는 아니다.

### 도감

- `GameCodex.jsx`는 존재하지만 아이템 도감 연동 여부는 미확정 상태다.
- 아이템 도감 확장은 UI 수정만으로 끝날지, 기존 아이템 API/데이터를 추가로 봐야 할지 분석이 필요하다.

---

## 작업 분리 원칙

이번 작업은 한 번에 다 밀지 않는다.

### Batch A: 모바일 HUD 개편

- 화면 배치
- 버튼 위치
- 조작 UX
- 모바일 기준 크기/간격 조정

### Batch B: 도감 확장

- `GameCodex` 구조 분석
- 아이템 데이터 소스 확인
- 아이템 도감 탭/리스트/상세 연동

이유:

- Batch A는 프론트 UI 중심이라 빠르게 검증 가능하다.
- Batch B는 데이터 구조와 UI 구조를 같이 봐야 해서 범위가 다르다.
