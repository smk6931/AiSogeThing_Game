# Title: 멀티플레이 최적화 작업 계획
Description: 실시간 이동, 스킬, 몬스터, 지역 조회의 병목을 줄이기 위한 단계별 작업 문서.
When-To-Read: 멀티플레이 성능 최적화, 소켓 트래픽 절감, 월드 지역 조회 부하 완화, 렌더 polling 정리 작업 전.
Keywords: multiplayer, websocket, optimization, region, polling, monster, player, skill
Priority: high

## 목표

- 플레이어 이동 패킷을 필요한 빈도로만 전송한다.
- 서버 브로드캐스트가 느린 소켓 하나 때문에 전체가 밀리지 않게 한다.
- 현재 지역 조회를 공용 상태로 통합해 중복 API 호출을 제거한다.
- 월드 관련 polling을 줄여 추후 몬스터/스킬 멀티 최적화의 기반을 만든다.

## 작업 순서

1. 이동 소켓 전송 경로 분리
- 로컬 움직임 반영과 서버 동기화 전송을 분리한다.
- 이동 패킷은 제한된 주기와 유의미한 변화가 있을 때만 전송한다.

2. 서버 브로드캐스트 구조 개선
- 브로드캐스트를 병렬 전송으로 전환한다.
- 실패한 연결은 정리 가능한 형태로 수집한다.

3. 현재 지역 조회 공용화
- 프론트에서 현재 GPS/동/파티션 정보를 한 곳에서 계산하고 캐시한다.
- `getCurrentRegion` 중복 호출을 제거한다.

4. 월드 polling 정리
- 파티션 경계, 오버레이, 미니맵이 각자 500ms/700ms polling 하지 않도록 조정한다.
- 기존 UI 표시 동작은 유지한다.

5. 검증
- 이동/채팅/스킬/몬스터 피격/지역 표시가 기존처럼 동작하는지 확인한다.
- 추후 작업 포인트를 남긴다.

## 추후 확장 포인트

- 몬스터 동기화를 전체 스냅샷 방송 대신 delta 이벤트 중심으로 전환
- 근접 플레이어/근접 몬스터 관심영역(AOI) 기반 전송
- 스킬 이벤트도 브로드캐스트 범위 제한과 배치 전송 적용

## 진행 기록

- 완료: 이동 패킷 중복 전송 완화
- 완료: 서버 브로드캐스트 병렬화
- 완료: 현재 지역 조회 공용화 및 중복 polling 제거
- 완료: AOI 기반 플레이어/스킬 전송
- 완료: 몬스터 delta 동기화
- 완료: 몬스터 delta의 플레이어별 가시 범위 분배
